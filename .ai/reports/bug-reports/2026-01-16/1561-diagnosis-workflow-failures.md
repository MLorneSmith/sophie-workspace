## Detailed Root Cause Analysis

### Issue 1: E2E Tests (Sharded) Failure

**Root Cause**: Duplicate `--cache-dir=.turbo` argument passed to Turbo

**Error**:
```
ERROR  the argument '--cache-dir <CACHE_DIR>' cannot be used multiple times
```

**Explanation**:
- The workflow runs `pnpm build --cache-dir=.turbo`
- The `pnpm build` script in `package.json` is defined as: `./.claude/statusline/build-wrapper.sh turbo build --cache-dir=.turbo`
- This results in `turbo build --cache-dir=.turbo --cache-dir=.turbo` which Turbo rejects

**Affected Files**:
- `package.json:17` - The build script
- `.github/workflows/e2e-sharded.yml:80` - Workflow passes `--cache-dir=.turbo`

**Fix Approach**: Either remove `--cache-dir=.turbo` from the workflow, or update `package.json` to use `build:raw` instead.

### Issue 2: Deploy to Production Failure

**Root Cause**: Permission denied on shell scripts during build

**Error**:
```
sh: 1: ./fix-build-permissions.sh: Permission denied
sh: 1: ./.claude/statusline/build-wrapper.sh: Permission denied
ELIFECYCLE  Command failed with exit code 126.
```

**Explanation**: The production deploy workflow doesn't preserve executable permissions on shell scripts. Exit code 126 indicates "Permission denied" when trying to execute a command.

**Affected Files**:
- `fix-build-permissions.sh` - Missing execute permission
- `.claude/statusline/build-wrapper.sh` - Missing execute permission
- `.github/workflows/production-deploy.yml` - Needs to set execute permissions

**Fix Approach**: Add `chmod +x` step before build in production workflow, or ensure git preserves executable bits.

### Issue 3: Build CI Docker Image Failure

**Root Cause**: CodeQL SARIF upload permission error

**Error**:
```
##[error]Resource not accessible by integration - https://docs.github.com/rest/actions/workflow-runs#get-a-workflow-run
```

**Explanation**: The workflow's GITHUB_TOKEN lacks permissions to upload SARIF files to the GitHub Security tab. The Trivy scan itself completed successfully; only the CodeQL integration failed.

**Affected Files**:
- `.github/workflows/docker-ci-image.yml` - Has `security-events: write` but may need additional permissions
- `.github/actions/trivy-scan/action.yml` - Uses `github/codeql-action/upload-sarif@v3`

**Fix Approach**: Either update workflow permissions or downgrade from `upload-sarif` action requirement.

- `.ai/alpha/progress/*.json`

**Fix Approach**: Either format these files with `pnpm format:fix`, or add `.ai/alpha/` to Biome's ignore list in `biome.json`.
