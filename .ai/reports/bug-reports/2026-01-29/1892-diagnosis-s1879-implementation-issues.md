## Issue A: RSC Payload Fetch Failure

### Error Message
```
Failed to fetch RSC payload for https://3000-iw2ke42k6k7b1t0jwqeiy.e2b.app/home/course.
Falling back to browser navigation. TypeError: Failed to fetch
```

### Diagnosis: NOT A BUG

**Root Cause**: This is expected behavior in E2B sandbox environments due to:
1. **Network latency** - Remote sandboxes have additional network hops
2. **Development server** - `next dev` is slower than production builds
3. **Cold start** - First navigation to routes triggers on-demand compilation
4. **RSC fetch timeout** - Client-side RSC fetches have timeouts that can be exceeded

**Impact**: Minimal - Next.js gracefully falls back to browser navigation (full page reload). The app still works correctly.

**Recommendation**:
- This is **not** a problem with the orchestrator or the S1879 implementation
- For better UX, add `loading.tsx` files to routes (e.g., `apps/web/app/home/[account]/course/loading.tsx`)
- In production with CDN, this issue will be much less common

## Issue B: Documentation Event Shows Error Icon

### Log Message
```
09:55:15 [orchestrator] ❌ Starting spec documentation generation via /alp...
```

### Diagnosis: BUG (FIXED)

**Root Cause**: The `documentation_start` event type was missing from the `validTypes` array in `.ai/alpha/scripts/ui/index.tsx`. This caused the event to fall back to `"error"` type, displaying ❌ instead of 📚.

**Fix Applied**: Added missing event types to the array:
```typescript
// Documentation generation event types (Bug fix: missing from validTypes)
"documentation_start",
"documentation_complete",
"documentation_failed",
```

**GitHub Issue**: #1892

## Issue C: Review Sandbox Creating Message

### Log Message
```
09:57:39 [orchestrator] 📦 Creating fresh review sandbox for dev server
```

### Diagnosis: NOT AN ERROR - EXPECTED BEHAVIOR

**Explanation**: This message uses 📦 (package) icon and blue color, which is the correct formatting for `review_sandbox_creating` events. The icon/color scheme is:
- 📦 blue = creating/starting
- ✅ green = success
- ❌ red = error

This message indicates normal operation - the orchestrator is creating a fresh review sandbox for you to access the dev server.

**Why it might look like an error**: The user may have expected only informational messages without icons during the completion phase. The 📦 icon is intentionally prominent to show sandbox lifecycle events.
