# Bug Diagnosis: Progress Poller Race Condition Causes Stale Feature Display

**ID**: ISSUE-1633
**Created**: 2026-01-20T17:00:00Z
**Reporter**: user
**Severity**: high
**Status**: confirmed
**Type**: bug (race condition)

## Summary

The UI progress display shows the wrong feature (e.g., `S1607.I4.F1` instead of `S1607.I5.F3`) due to a race condition in the progress polling system. Multiple pollers can write to the same progress file concurrently, causing stale feature data to overwrite current data.

## Observed Behavior

- UI shows "Quick Actions Panel 0/10" but actual work is on "Upcoming Sessions Display"
- The `feature` field in progress file shows random OLD features (I1.F1, I4.F1, etc.)
- The `current_task` and `recent_output` fields show CORRECT current work
- Task completion count doesn't update in UI

## Root Cause

**Race condition in `startProgressPolling()` in `.ai/alpha/scripts/lib/progress.ts:320-394`**

```javascript
const poll = async () => {
  while (isPolling) {
    await sandbox.commands.run(...);  // Takes up to 5 seconds
    writeUIProgress(..., feature);    // Writes with CLOSURE's feature
  }
};

return {
  stop: () => { isPolling = false; },  // Doesn't wait for poll to finish!
  ...
};
```

**The problem:**
1. Feature A's `poll()` starts `await sandbox.commands.run()` (5s timeout)
2. Feature A completes, `progressPoller.stop()` sets `isPolling = false`
3. Feature B starts, creates NEW poller with NEW feature
4. Feature A's `await` completes, writes to progress file with Feature A's data
5. Feature B's poller writes with Feature B's data
6. Feature A's loop iteration continues (isPolling not checked until next iteration), writes AGAIN
7. **Result: Feature A's data intermittently overwrites Feature B's data**

## Fix Approach

**Option 1: Make stop() await current poll iteration**
```javascript
let currentPollPromise: Promise<void> | null = null;

const poll = async () => {
  while (isPolling) {
    currentPollPromise = (async () => {
      const result = await sandbox.commands.run(...);
      if (!isPolling) return; // Early exit if stopped during await
      writeUIProgress(...);
    })();
    await currentPollPromise;
    currentPollPromise = null;
  }
};

return {
  stop: async () => {
    isPolling = false;
    if (currentPollPromise) await currentPollPromise;
  },
  ...
};
```

**Option 2: Add mutex/lock around writeUIProgress**
```javascript
const writeLock = new Mutex(); // Or use a simple flag

// In writeUIProgress
await writeLock.acquire();
try {
  fs.writeFileSync(filePath, JSON.stringify(uiProgress));
} finally {
  writeLock.release();
}
```

**Option 3: Check feature assignment before writing**
```javascript
// In poll()
if (!isPolling || instance?.currentFeature !== feature?.id) return;
writeUIProgress(...);
```
