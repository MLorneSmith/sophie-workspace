## Key UI Patterns Across All 10 Top Candidates

Based on this research, the following UI patterns appear consistently across design-lauded SaaS tools:

### 1. Command Palette (Cmd+K / Ctrl+K)
- **Found in:** Linear, Notion, Vercel, Raycast, Figma
- **Why:** Power users demand keyboard-driven navigation
- **Implementation tips:**
  - Search by fuzzy matching
  - Show keyboard shortcuts in results
  - Categorize results (Actions, Pages, Settings)
  - Keep it accessible from anywhere in the app

### 2. Canvas-Based Editing
- **Found in:** Figma, Miro, Gamma, Beautiful.ai, Tome
- **Why:** Visual tools require visual editing environments
- **Implementation tips:**
  - Minimal chrome (toolbars, sidebars) to maximize canvas space
  - Smooth pan/zoom with scroll wheel and gestures
  - Contextual toolbars that appear near selected objects
  - Keyboard shortcuts for common actions (delete, copy/paste, undo/redo)

### 3. Real-Time Collaboration Indicators
- **Found in:** Figma, Miro, Gamma, Tome
- **Why:** Modern work is collaborative
- **Implementation tips:**
  - Show user cursors with names and colors
  - Presence indicators in avatars (who's viewing)
  - Live typing indicators
  - Conflict resolution UI when multiple users edit same element

### 4. Right-Side Contextual Panels
- **Found in:** Figma, Linear, Miro, Webflow, Gamma
- **Why:** Non-intrusive way to show properties/details without losing context
- **Implementation tips:**
  - Panel content changes based on selection
  - Collapsible sections with clear hierarchy
  - Preview changes in real-time as panel options are adjusted
  - Closeable via Esc key or clicking outside

### 5. Template Libraries with Previews
- **Found in:** Gamma, Beautiful.ai, Tome, Miro, Notion
- **Why:** Reduce time-to-value, help users get started quickly
- **Implementation tips:**
  - Visual grid with hover previews
  - Categories and filters for discoverability
  - "Use template" CTA is prominent
  - Show template complexity (number of slides, estimated time)

### 6. Progressive Disclosure
- **Found in:** All 10 top candidates
- **Why:** Don't overwhelm users — show simple first, reveal complexity on demand
- **Implementation tips:**
  - Show common options by default
  - "Advanced" sections or "More options" for power users
  - Tooltips and inline help for complex features
  - Empty states that guide users to next action

### 7. Clean Sidebar Navigation
- **Found in:** Notion, Linear, Vercel, Gamma, Airtable
- **Why:** Users need to navigate between content areas easily
- **Implementation tips:**
  - Collapsible to save space
  - Visual hierarchy with icons and text
  - Active state clearly indicated
  - Drag-reorder for customization (when applicable)
  - Search/filter for long lists

### 8. Multi-View Data Display
- **Found in:** Airtable, Miro, Linear, Notion
- **Why:** Same content, different perspectives for different tasks
- **Implementation tips:**
  - View switcher with clear icons/names
  - State persists per view
  - Shareable view URLs
  - Custom view creation

### 9. AI-Assisted Creation
- **Found in:** Gamma, Beautiful.ai, Tome, Figma (AI features)
- **Why:** AI reduces friction in creative work
- **Implementation tips:**
  - Prompt input is prominent and approachable
  - Show multiple options/variations
  - "Regenerate" with slight variations
  - Allow manual override (AI suggestion, not AI mandate)
  - Progress indicators during generation

### 10. Export/Share Modal with Clear Options
- **Found in:** All presentation tools + Miro, Figma
- **Why:** Sharing is a core action for collaborative tools
- **Implementation tips:**
  - Clear format options (PDF, PPTX, link, embed)
  - Permission settings for sharing (view, comment, edit)
  - Copy-to-clipboard buttons for URLs
  - Preview of what will be shared
