# SaaS in-app dashboard / home-screen inspiration (5 examples)

Focus: **in-app home screens after login** that combine *creation* + *project management* + clear next-step guidance.

---

## 1) Pitch — Presentation workspace dashboard

**Company + product:** Pitch (collaborative presentation software)

**Screenshot / reliable source:** Pitch Help Center (“Get to know your Pitch dashboard”) — includes multiple annotated in-app dashboard screenshots.
- Dashboard overview screenshot: https://downloads.intercomcdn.com/i/o/alrusdv4/1647729406/3d1ff162524a4a68171b8b6c4114/dashboard.png
- “Create” section screenshot: https://downloads.intercomcdn.com/i/o/alrusdv4/1647732342/61f9051273acebaf4b8717a19833/dashboard-create.png
- Source page: https://help.pitch.com/en/articles/8038180-get-to-know-your-pitch-dashboard

**What makes it great (specific UI/design notes):**
- **Clear primary actions at the top** (start from template / start with AI / import PPT) — obvious “what should I do next?”
- **Strong information hierarchy**: creation options first, then recents/activity, then library.
- **Creation + management on one screen**: you can immediately create new work *and* jump back into recent presentations.
- **Personalization**: dashboard is “unique to each member” and recents can filter (e.g., *By me* vs *By everyone*).

**Relevance to SlideHeroes:**
- SlideHeroes also lives in the “presentation creation” category, so Pitch’s dashboard patterns map directly: **New deck CTA + templates + AI entry point + recents**.
- The “import PPT” option is a great reminder: many small consultancies start from existing client decks.

## 2) Notion — Home dashboard (widgets for recents, tasks, calendar)

**Company + product:** Notion (workspace/wiki/docs)

**Screenshot / reliable source:** Thomas Frank’s deep dive on **Notion Home** includes full-width screenshots of the in-app Home dashboard.
- Example Home screenshot (direct image URL on page): https://thomasjfrank.com/wp-content/uploads/2024/05/CleanShot-2024-05-30-at-13.02.11@2x.jpg
- Source page: https://thomasjfrank.com/notion-home-everything-you-need-to-know/

**What makes it great (specific UI/design notes):**
- **Modular widget layout** (upcoming events, tasks, recently visited, suggested pages) gives at-a-glance context without feeling heavy.
- **“Return to where you were” support** via recents + suggested items reduces friction.
- **Dashboard feels personal** (content is user-specific; widgets reflect the user’s activity).
- **Good empty-state behavior** (e.g., connect calendar, show suggested templates) — avoids the “blank screen problem.”

**Relevance to SlideHeroes:**
- SlideHeroes can borrow the **widget concept**: e.g., *Recent presentations*, *Continue editing*, *Drafts needing review*, *Client-specific folders*, *Suggested next actions*.
- Notion’s Home is a strong reference for **lightweight personalization** without needing complex analytics.

## 3) Figma — Files/Drafts home area (teams, drafts, recent)

**Company + product:** Figma (design/collaboration platform)

**Screenshot / reliable source:** Figma Forum thread with user-posted screenshots of the in-app “Drafts”/“Drafts to move” area (part of the logged-in file browser / home experience).
- Screenshot (direct image URL): https://uploads-us-west-2.insided.com/figma-en/attachment/9534e8f1680171c0ceca0805988dcd0eb6135551.png
- Additional screenshot (direct image URL): https://uploads-us-west-2.insided.com/figma-en/attachment/0f3d0b2407ab5e23fa7f7137bcbf687c353c0934.png
- Source page: https://forum.figma.com/ask-the-community-7/drafts-moved-to-teams-11421

**What makes it great (specific UI/design notes):**
- **Left-nav + main content split** keeps orientation stable (teams/projects/sections on the left; files in the main panel).
- **Strong “place” model**: drafts, teams, starred, and projects give users predictable buckets for work.
- **Scales from solo to team**: the same home surface supports personal drafts and collaborative spaces.

**Relevance to SlideHeroes:**
- SlideHeroes can adopt Figma’s **workspace + drafts** mental model: personal drafts vs client/team spaces.
- The dashboard can emphasize **structure + retrieval**: quick filters (recent, starred, by client) and consistent navigation to reduce “where is that deck?” pain.

## 4) Canva — Home dashboard (create + templates + recents)

**Company + product:** Canva (visual design & presentation creation)

**Screenshot / reliable source:** YouTube walkthrough focusing specifically on the **Canva homepage/dashboard** (in-app after login).
- Video: https://www.youtube.com/watch?v=ZZr9M8QIA5U  (shows the logged-in Canva home screen; use as a visual reference for layout + modules)

**What makes it great (specific UI/design notes):**
- **Huge “Create” affordance** plus template shortcuts make starting fast.
- **Template-driven onboarding**: users who don’t know where to begin can pick a format and go.
- **Recents + search** support returning users (find past work quickly).
- **Content discovery is integrated** (suggestions, formats, and templates live on the same home surface as your existing projects).

**Relevance to SlideHeroes:**
- Canva is a direct adjacent category: **presentations + templates + brand assets**.
- SlideHeroes can borrow the “new deck” module pattern: *Start with AI*, *Start from template*, *Start from your brand kit*, plus **recents** underneath.

## 5) Miro — Dashboard + “New board” starting point

**Company + product:** Miro (whiteboard / collaborative canvas)

**Screenshot / reliable source:** YouTube tutorial that starts from the **Miro dashboard** (templates + new board + recent boards).
- Video: https://www.youtube.com/watch?v=1EXHLhoQRas (begins with the logged-in dashboard in typical tutorials; use as reference for “create + recents + templates” composition)

**What makes it great (specific UI/design notes):**
- **Creation-first home**: dashboard flow naturally funnels users into “create new board” while still showing recent work.
- **Template entry point** is prominent (helps users who don’t have a plan yet).
- **Recent boards** makes continuation effortless.

**Relevance to SlideHeroes:**
- SlideHeroes can mimic the “canvas product” approach: a dashboard that quickly branches into **blank deck**, **template**, or **guided flow** (e.g., “Client update deck”, “Proposal deck”, “Workshop readout”).
- Miro’s balance between *starting new* and *continuing existing* is exactly the dashboard tension SlideHeroes needs to solve.

## Patterns worth stealing for SlideHeroes (cross-cutting)

- **Top-of-screen primary actions**: “Create new presentation” + 2–3 secondary choices (Template / AI / Import).
- **Recents with metadata**: last edited, client/workspace, status (draft, shared, exported).
- **Light personalization**: “Continue where you left off” + “Suggested next step” (e.g., finish outline, apply theme, generate speaker notes).
- **Empty states that teach**: if no decks yet, show 3 clickable example outcomes + short promise + one primary CTA.
- **Workspace structure**: *Personal drafts* vs *Client folders* vs *Team space*.
