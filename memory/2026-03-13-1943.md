The problem is it was **missing `task_key`** — the condition code requires a `task_key` to track cooldown state per-PR:

```python
# From not_in_cooldown.py
task_key = config.get("task_key") or context.get("task_key")

if not task_key:
    return NodeResult(
        status="error",
        error="task_key required for not_in_cooldown condition",
    )
```

Without a unique `task_key`, the condition can't track whether a specific PR is in cooldown or not. It would try to use all PRs with the same cooldown state, which would block all PRs after one triggers.

**The fix** (which I applied) was to add:
```json
"task_key": "review-response:{{trigger.repo}}:{{trigger.number}}"
```

This creates a unique key per PR like `review-response:slideheroes/roadmap:2`, so each PR has its own cooldown tracking.

The workflow was failing at the cooldown check because it was throwing an error about missing `task_key`, then skipping downstream nodes (including spawning Neo).
