## [17:00] Memory System — Architecture Decisions Log

**Key decisions made during implementation:**
- MiniMax embo-01 embedding model incompatible with Mem0 OSS (non-standard API format using `texts` instead of `input`)
- Anthropic has no embedding model — recommends Voyage AI
- Chose OpenAI text-embedding-3-small for embeddings ($0.02/1M tokens)
- gpt-4o-mini for Mem0 extraction LLM (MiniMax uses Anthropic API format, incompatible with Mem0 OSS openai provider)
- OpenAI `openai` provider in openclaw.json points to ccproxy, NOT real OpenAI — had to explicitly set api_key in Mem0 oss config
- sqlite3 native bindings missing after plugin install — had to `npm install sqlite3` in openclaw dir AND symlink to jiti paths
- ChatGPT Pro does NOT include API/embeddings access — separate billing required
- Official Mem0 plugin chosen over v2 community plugin for stability; sleep mode built as custom script instead
- `openclaw mem0` CLI spins up separate process that doesn't share gateway's in-memory vector store — CLI shows 0 memories while gateway has 40+

**Lesson:** When configuring Mem0 OSS, check that the `openai` provider in openclaw.json actually points to real OpenAI. Our setup routes it through ccproxy which doesn't support embeddings.
