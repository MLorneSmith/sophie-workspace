## [16:15] ACP Session Bug — Root Cause Found & Patched

**Bug:** `claude-agent-acp` v0.19.2 `loadSession()` passes acpx session UUIDs as `--resume <id>` to Claude Code SDK, but those UUIDs don't exist in Claude Code's session store (`~/.claude/projects/`). Claude Code exits with code 1, causing `ACP_TURN_FAILED`.

**Root cause chain:**
1. `sessions_spawn` → OpenClaw ACP runtime → acpx `ensureSession` + `prompt --session`
2. acpx calls `session/load` on `claude-agent-acp`
3. `loadSession()` calls `createSession({}, {resume: params.sessionId})`
4. SDK passes `--resume <acpx-uuid>` to Claude Code CLI
5. Claude Code can't find `<uuid>.jsonl` → exits code 1
6. Error surfaces as "Query closed before response received" / `ACP_TURN_FAILED`

**Why `acpx exec` worked:** One-shot, no `session/load`, no `--resume`.

**Patch applied to** `/home/ubuntu/.npm-global/lib/node_modules/@zed-industries/claude-agent-acp/dist/acp-agent.js`:
1. `loadSession()` — checks `listSessions()` before resuming; falls back to fresh session with same ID
2. `createSession()` — accepts `sessionId` in `creationOpts` for non-resume path

**Backup:** `acp-agent.js.bak` in same directory.
**⚠️ Patch will be overwritten on `claude-agent-acp` update.** 

**Tested:** Both `acpx prompt --session` and `sessions_spawn(runtime="acp")` now work.
**Upstream issue:** Couldn't file (PAT lacks permission on zed-industries/claude-agent-acp). Mike to file manually.

## [18:00] OOM Crash — Root Cause & Fix

**What happened:** 4 concurrent ACP Claude Code sessions spawned by neo-ci-fix.py consumed ~900MB+ on top of the ~1.8GB baseline (gateway + system). Hit OOM killer on the 3.7GB c7i-flex.large instance. System went down.

**Fix:** Rewrote `process-spawn-queue.py` with:
1. **Serial execution** — only 1 task at a time, runs synchronously (waits for completion)
2. **Memory gate** — checks `/proc/meminfo`, skips if <500MB available
3. **Concurrency guard** — checks for existing `acpx exec` processes before spawning
4. **Remaining tasks stay in queue** for next cron cycle (10-30 min)

**Memory budget:** ~200MB per ACP session. With 2.4GB available headroom, safe for 1 session at a time.

**Detection scripts updated:** `queue_spawn()` now triggers `process-spawn-queue.py` in background (Popen), which runs the NEXT task synchronously.
