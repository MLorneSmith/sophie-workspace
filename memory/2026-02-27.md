## [21:30] Data Quality Discussion — Profile Sparse Data (#517)

**Individual Profile:**
- Goal is adapting style/approach, not completeness
- Minimum threshold: role + seniority + industry = sufficient to be useful
- Below threshold → prompt user for manual input
- Binary: sufficient vs insufficient (not a confidence spectrum)

**Company Profile:**
- Two dimensions: **completeness** (how many fields) and **confidence/verifiability** (how reliable)
- Dangerous quadrant: complete but low confidence — looks good but contains inferences user might present as fact
- Summary-level indicator is enough for beta ("Profile based on 4 verified sources" vs "partially inferred")

**Deferred to v2:**
- **Field-level confidence** across all presentation data — a major feature. Every data point tagged with confidence + source. Enables "verify key facts" workflows before presenting.
