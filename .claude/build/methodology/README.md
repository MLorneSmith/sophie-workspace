# AAFD v2.0 Methodology Implementation

## Overview

This directory contains the core implementation of the AI-Assisted Feature Development (AAFD) v2.0 methodology for SlideHeroes development.

## Directory Structure

- `methodology-design/` - Core methodology design documents
- `prompt-library/` - XML-based prompt templates for AI interactions
- `contexts/` - Context management system for development sessions
- `templates/` - Reusable templates for issues, contexts, and workflows
- `workflows/` - Development workflow documentation
- `roles/` - Role-specific guidance for different types of development
- `docs/` - Supporting documentation and guides

## Quick Start

1. **Feature Planning**: Use prompts in `prompt-library/feature-planning.xml`
2. **Context Loading**: Use templates in `contexts/session-templates/`
3. **GitHub Integration**: Use templates in `templates/github/`

## Methodology Phases

1. **Ideation & PRD Creation** - Transform ideas into structured requirements
2. **Technical Chunking & Analysis** - Break requirements into implementation chunks
3. **Story Refinement & Estimation** - Create implementable user stories
4. **Sprint Planning & Execution** - Plan and execute development sprints
5. **Review & Retrospective** - Validate and improve the process
