# Migration Strategies

## Overview

SlideHeroes uses a complex multi-schema migration system:

1. **Supabase Migrations** - For the `public` schema (MakerKit SaaS infrastructure)
2. **Payload CMS Migrations** - For the `payload` schema (content management)
3. **UUID Table Management** - For managing Payload CMS relationships
4. **PowerShell Orchestration** - For coordinating complex migrations

This document outlines our strategies for managing database migrations across all systems.

## Supabase Migrations

### Migration Structure

Supabase migrations are stored in `apps/web/supabase/migrations` and follow this naming convention:

```
YYYYMMDDHHMMSS_descriptive_name.sql
```

Example:
```
20241201120000_create_course_progress_table.sql
```

### Creating Migrations

#### 1. Generate Migration from Diff

```bash
pnpm --filter web supabase db diff create_new_feature
```

This compares your local schema with the remote database and generates a migration.

#### 2. Manual Migration Creation

For complex changes, create migrations manually:

```bash
pnpm --filter web supabase migration new create_accounts_table
```

### Migration Content for MakerKit

SlideHeroes uses MakerKit patterns with RLS policies:

```sql
-- Create accounts table
CREATE TABLE public.accounts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  slug TEXT UNIQUE,
  type TEXT NOT NULL DEFAULT 'personal',
  picture_url TEXT,
  primary_owner_user_id UUID REFERENCES auth.users(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable RLS
ALTER TABLE public.accounts ENABLE ROW LEVEL SECURITY;

-- Create RLS policies
CREATE POLICY "Users can view accounts they belong to" ON public.accounts
  FOR SELECT USING (
    auth.uid() IN (
      SELECT user_id FROM public.accounts_memberships 
      WHERE account_id = accounts.id
    )
  );
```

### Applying Migrations

Apply migrations during development:

```bash
pnpm supabase:web:reset
```

For production, use the orchestration scripts to ensure proper coordination.

## Payload CMS Migrations

### Migration Structure

Payload migrations are stored in `apps/payload/src/migrations` and include both:
- TypeScript migration files
- SQL migration files generated by Payload

```
YYYYMMDDHHMMSS_descriptive_name.ts
```

### Creating Migrations

Generate a migration after modifying your Payload config:

```bash
cd apps/payload
npx payload migrate:create
```

This creates migration files in both TypeScript and SQL formats.

### Migration Content for Course System

Payload migrations handle the complex course content structure:

```typescript
import type { MigrateUpArgs, MigrateDownArgs } from '@payloadcms/db-postgres'
import { sql } from 'drizzle-orm'

export async function up({ payload, req }: MigrateUpArgs): Promise<void> {
  await payload.db.execute(sql`
    CREATE TABLE IF NOT EXISTS "payload"."course_lessons" (
      "id" varchar PRIMARY KEY NOT NULL,
      "title" varchar NOT NULL,
      "slug" varchar NOT NULL,
      "content" varchar,
      "course_id" varchar,
      "bunny_video_id" varchar,
      "todo" varchar,
      "order_index" numeric,
      "status" varchar DEFAULT 'draft',
      "updated_at" timestamp(3) with time zone DEFAULT now() NOT NULL,
      "created_at" timestamp(3) with time zone DEFAULT now() NOT NULL
    );
  `)
}

export async function down({ payload, req }: MigrateDownArgs): Promise<void> {
  await payload.db.execute(sql`DROP TABLE IF EXISTS "payload"."course_lessons";`)
}
```

### Applying Migrations

Apply Payload migrations:

```bash
cd apps/payload
npx payload migrate
```

## PowerShell Orchestration Scripts

SlideHeroes uses PowerShell scripts for complex migration orchestration in `/scripts/orchestration/`:

### Main Orchestration Scripts

#### Complete Migration Process

```powershell
# Run complete database reset and migration
./scripts/orchestration/use-fixed-verification.ps1
```

#### Phase-based Migration

```powershell
# Setup phase
./scripts/orchestration/phases/setup.ps1

# Processing phase
./scripts/orchestration/phases/processing.ps1

# Loading phase
./scripts/orchestration/phases/loading.ps1

# Relationship repair
./scripts/orchestration/phases/relationship-repair.ps1
```

### UUID Table Management

Critical for Payload CMS relationships:

```powershell
# Create UUID tables for all collections
./scripts/orchestration/utils/verification.ps1 -CreateUUIDTables

# Verify UUID table integrity
./scripts/orchestration/utils/verification.ps1 -VerifyUUIDTables
```

### Content Migration

For migrating content between environments:

#### Progressive Migration

```powershell
./scripts/remote-migration/content/migrate-content-progressive.ps1 -Environment Production
```

Options:
- `-SkipCore` - Skip core MakerKit tables
- `-SkipPosts` - Skip blog posts
- `-SkipDocumentation` - Skip documentation
- `-SkipCourses` - Skip course content
- `-SkipQuizzes` - Skip quiz data
- `-SkipSurveys` - Skip survey data
- `-SkipDownloads` - Skip download files

#### Direct SQL Transfer

For large datasets:

```powershell
./scripts/remote-migration/utils/direct-sql-transfer.ps1 -TableName courses -BatchSize 100
```

### Verification Scripts

Comprehensive verification after migrations:

```powershell
# Verify all content integrity
./scripts/remote-migration/content/verify-remote-content.ps1 -VerifyAll

# Verify specific collections
./scripts/remote-migration/content/verify-remote-content.ps1 -VerifyCourses -VerifyQuizzes

# Verify relationship integrity
./scripts/orchestration/utils/verification.ps1 -VerifyRelationships
```

## Best Practices

### 1. UUID Table Consistency

Ensure UUID tables exist for all Payload collections:

```powershell
# Create missing UUID tables
./scripts/orchestration/utils/verification.ps1 -CreateUUIDTables

# Verify UUID table integrity
./scripts/orchestration/utils/verification.ps1 -VerifyUUIDTables
```

### 2. Relationship Integrity

Maintain relationship consistency across schemas:

```sql
-- Create relationship tracking views
CREATE OR REPLACE VIEW payload.course_lesson_relationships AS
SELECT 
  cl.id as lesson_id,
  cl.title as lesson_title,
  cl.course_id,
  c.title as course_title
FROM payload.course_lessons cl
LEFT JOIN payload.courses c ON c.id = cl.course_id;
```

### 3. Incremental Migration

Use progressive migration for large datasets:

```powershell
# Migrate in batches
./scripts/remote-migration/content/migrate-content-progressive.ps1 -BatchSize 50 -MaxConcurrency 3
```

### 4. Verification at Each Step

Always verify after each migration phase:

```powershell
# Verify after each phase
./scripts/orchestration/phases/setup.ps1
./scripts/orchestration/utils/verification.ps1 -VerifySetup

./scripts/orchestration/phases/processing.ps1
./scripts/orchestration/utils/verification.ps1 -VerifyProcessing
```

### 5. Rollback Strategy

Maintain rollback capabilities:

```powershell
# Create backup before migration
./scripts/database-reset/backup-remote-db.sh

# Emergency rollback if needed
./scripts/database-reset/emergency-rollback.sh
```

## Migration Workflow

### Development Workflow

1. **Schema Changes**:
   - Modify Payload config for content schema
   - Update Supabase schema for SaaS tables
   - Generate migrations for both systems

2. **Local Testing**:
   ```bash
   # Reset and test locally
   pnpm supabase:web:reset
   cd apps/payload && npx payload migrate
   ```

3. **Verification**:
   ```powershell
   ./scripts/orchestration/utils/verification.ps1 -VerifyAll
   ```

### Production Deployment

1. **Backup**:
   ```powershell
   ./scripts/database-reset/backup-remote-db.sh
   ```

2. **Migration**:
   ```powershell
   ./scripts/orchestration/use-fixed-verification.ps1
   ```

3. **Verification**:
   ```powershell
   ./scripts/remote-migration/content/verify-remote-content.ps1 -VerifyAll
   ```

## Troubleshooting

### Failed Migrations

For Payload migration failures:

```bash
# Check migration status
cd apps/payload
npx payload migrate:status

# Reset and retry
npx payload migrate:reset
npx payload migrate
```

### UUID Table Issues

Fix UUID table inconsistencies:

```powershell
# Diagnose UUID table issues
./scripts/orchestration/utils/verification.ps1 -DiagnoseUUIDTables

# Repair UUID tables
./scripts/orchestration/utils/verification.ps1 -RepairUUIDTables
```

### Relationship Corruption

Repair broken relationships:

```powershell
# Repair all relationships
./scripts/orchestration/phases/relationship-repair.ps1

# Repair specific collection relationships
./scripts/orchestration/phases/relationship-repair.ps1 -Collections courses,quizzes
```

### Performance Issues

Optimize migration performance:

```powershell
# Use parallel processing
./scripts/remote-migration/content/migrate-content-progressive.ps1 -MaxConcurrency 5 -BatchSize 25

# Monitor performance
./scripts/orchestration/utils/performance-logging.ps1
```
