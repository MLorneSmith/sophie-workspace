# Document Feature

Generate comprehensive technical documentation for implemented features by analyzing code changes, specifications, and review reports. Creates documentation in `.ai/ai_docs/context-docs/features/` with YAML frontmatter for automatic discovery by the conditional docs system.

## Instructions

IMPORTANT: You're creating technical documentation for an implemented feature.
IMPORTANT: This is about documenting what was built, how it works, and how to use it.
IMPORTANT: Documentation will be automatically discoverable via the conditional docs system using YAML frontmatter.
You're creating comprehensive, well-structured documentation that helps future developers understand and use this feature.

### 1. Fetch Implementation Issue from GitHub

**Fetch the implemented feature issue**:
```bash
# Extract issue number from $ARGUMENTS and fetch the issue
gh issue view <issue-number> \
  --repo MLorneSmith/2025slideheroes \
  --json body,title,labels,number,url,comments \
  --jq '{body: .body, title: .title, labels: [.labels[].name], number: .number, url: .url, comments: [.comments[] | {author: .author.login, body: .body, createdAt: .createdAt}]}'
```

**Validate it's documentable**:
- Check for "implemented" label OR "review-passed" label
- If not ready for documentation, inform user the feature needs to be implemented first
- Extract spec file path from issue body (look for file paths in `.ai/specs/`)

### 6. Analyze Implementation

**Analyze git changes**:
```bash
# Get overview of changes
git diff origin/main --stat

# Get list of all changed files
git diff origin/main --name-only

# For significant changes (>50 lines), get detailed diff
# to understand implementation details
for file in $(git diff origin/main --name-only); do
  lines_changed=$(git diff origin/main --numstat "$file" | awk '{print $1+$2}')
  if [ "$lines_changed" -gt 50 ]; then
    echo "=== Detailed diff for $file (${lines_changed} lines changed) ==="
    git diff origin/main "$file"
  fi
done
```

**Read specification** (if exists):
- Read the spec file to understand:
  - Original requirements and goals
  - Expected functionality
  - Success criteria
  - Design decisions
- Use this to frame the documentation around what was requested vs what was built
- Note any deviations, enhancements, or future considerations

**Read review report** (if exists):

# Check if review report exists
if [ -f "${reviewReportPath}" ]; then

### 7. Process Screenshots

**Find screenshots** (check in priority order):

1. **From review report**: If review was done, screenshots are in `./reports/reviews/${issueNumber}/screenshots/`
2. **From implementation artifacts**: Check git history or implementation comments for screenshot references
3. **Manual capture**: If no screenshots exist and feature is UI-related, consider capturing them now

**Copy and organize screenshots**:
```bash
# Create target directory for feature screenshots
mkdir -p "${screenshotsTargetDir}"

# Check if review screenshots exist
if [ -d "${screenshotsSourceDir}" ] && [ "$(ls -A ${screenshotsSourceDir})" ]; then
  # Copy screenshots with descriptive names
  for screenshot in "${screenshotsSourceDir}"/*.png; do
    if [ -f "$screenshot" ]; then
      filename=$(basename "$screenshot")
      cp "$screenshot" "${screenshotsTargetDir}/${filename}"
      echo "Copied: ${filename}"
    fi
  done

  echo "Screenshots copied to: ${screenshotsTargetDir}"
else
  echo "No review screenshots found. Manual capture may be needed for UI features."
fi
```

**Screenshot naming convention**:
- Use descriptive names: `01_overview.png`, `02_login-flow.png`, `03_settings-panel.png`
- Number screenshots in logical sequence
- Keep names lowercase with hyphens

## Validation Commands

Before finalizing documentation:

```bash
# Validate documentation file was created
test -f "${documentationPath}" || echo "ERROR: Documentation file not created at ${documentationPath}"

# Validate YAML frontmatter exists and is properly formatted
if head -n 50 "${documentationPath}" | grep -q "^---$"; then
  echo "✓ YAML frontmatter found"

  # Extract and validate YAML frontmatter
  frontmatter=$(sed -n '/^---$/,/^---$/p' "${documentationPath}" | head -n -1 | tail -n +2)

  # Check required fields
  echo "$frontmatter" | grep -q "^title:" || echo "WARNING: Missing 'title' in frontmatter"
  echo "$frontmatter" | grep -q "^category:" || echo "WARNING: Missing 'category' in frontmatter"
  echo "$frontmatter" | grep -q "^tags:" || echo "WARNING: Missing 'tags' in frontmatter"
  echo "$frontmatter" | grep -q "^status:" || echo "WARNING: Missing 'status' in frontmatter"
  echo "$frontmatter" | grep -q "^github_issue:" || echo "WARNING: Missing 'github_issue' in frontmatter"
else
  echo "ERROR: Missing or invalid YAML frontmatter"
fi

# Validate markdown syntax (if markdownlint is available)
if command -v markdownlint >/dev/null 2>&1; then
  markdownlint "${documentationPath}" || echo "WARNING: Markdown linting issues found"
else
  echo "INFO: markdownlint not available, skipping markdown validation"
fi

# Validate screenshot paths exist
doc_dir=$(dirname "${documentationPath}")
grep -oP '!\[.*?\]\(\K[^)]+' "${documentationPath}" 2>/dev/null | while read -r img_path; do
  # Handle relative paths
  if [[ "$img_path" == ../* ]]; then
    full_path="${doc_dir}/${img_path}"
  else
    full_path="${img_path}"
  fi

  if [ -f "$full_path" ]; then
    echo "✓ Screenshot exists: $img_path"
  else
    echo "WARNING: Screenshot not found: $img_path (resolved to: $full_path)"
  fi
done

# Validate internal documentation links
grep -oP '\]\(\K[^)]+(?=\))' "${documentationPath}" 2>/dev/null | grep -v '^http' | grep '\.md' | while read -r link; do
  # Handle relative paths
  if [[ "$link" == ../* ]] || [[ "$link" == ./* ]]; then
    full_path="${doc_dir}/${link}"
  else
    full_path="${link}"
  fi

  if [ -f "$full_path" ]; then
    echo "✓ Linked doc exists: $link"
  else
    echo "WARNING: Linked documentation not found: $link (resolved to: $full_path)"
  fi
done

# Validate screenshots directory
if [ -d "${screenshotsTargetDir}" ]; then
  screenshot_count=$(find "${screenshotsTargetDir}" -type f -name "*.png" 2>/dev/null | wc -l)
  echo "✓ Screenshots directory exists with ${screenshot_count} screenshots"
else
  echo "INFO: No screenshots directory created (may not be needed for non-UI features)"
fi

# Verify GitHub issue was updated
gh issue view <issueNumber> \
  --repo MLorneSmith/2025slideheroes \
  --json labels \
  --jq '.labels[].name' 2>/dev/null | grep -q 'documented' || echo "WARNING: 'documented' label not added to issue"
```
