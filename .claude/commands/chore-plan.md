# Chore Implementation Planning

Create a detailed implementation plan from an existing chore issue (created by `/chore`). This command focuses on **detailed solution design and implementation strategy**, building upon the basic chore plan with thorough research and step-by-step guidance.

## Prerequisites

**REQUIRED**: A chore issue from the `/chore` command.

If you don't have a chore issue:
1. Run `/chore` first to create the basic chore plan
2. Then return here with the chore issue number

## Instructions

IMPORTANT: You're creating a **detailed implementation plan** for a chore that has already been defined.
IMPORTANT: Use sub-agents to research technologies, packages, and best practices before designing the solution.
IMPORTANT: The chore has already been scoped - your job is to design the detailed implementation approach.

### 1. Fetch and Validate Chore Issue

**Fetch the chore issue from GitHub**:
```bash
# Extract issue number from $ARGUMENTS and fetch chore with comments
gh issue view <issue-number> \
  --repo MLorneSmith/2025slideheroes \
  --json body,title,labels,number,comments \
  --jq '{body: .body, title: .title, labels: [.labels[].name], number: .number, comments: [.comments[] | {author: .author.login, body: .body, createdAt: .createdAt}]}'
```

**Validate it's a chore issue**:
- Check for "Chore:" prefix in title OR "type:chore" label
- If not a chore issue, instruct user to run `/chore` first
- If chore is missing key information, request user update it before planning

### 2. Validate Chore Quality

Before creating a detailed plan, ensure the chore contains:
- [ ] Chore description clearly defined
- [ ] Chore type identified (maintenance, code-update, documentation, tooling)
- [ ] Impact analysis documented
- [ ] Basic step-by-step tasks outlined
- [ ] Validation commands listed

If chore is incomplete, ask user to update the chore issue before proceeding.

### 3. Extract Key Information

Parse the chore and extract:
```typescript
const choreIssueNumber = '[issue-number]'; // From $ARGUMENTS
const choreTitle = '[Brief description for plan title]'; // e.g., "Update Next.js to v17"
const choreType = '[maintenance|code-update|documentation|tooling]'; // From labels or content
const priority = '[critical|high|medium|low]'; // From priority:* label
const affectedAreas = ['area1', 'area2']; // From content or area:* labels
```

### 4. Research Phase - Use Sub-Agents

**CRITICAL**: Before designing the implementation, use specialized agents to research relevant technologies and best practices.

#### 4a. Technology Documentation Research (context7-expert)

For any packages, tools, or technologies mentioned in the chore, use the context7-expert agent:

```
Task: context7-expert
Prompt: "Research documentation for [technology/package] focusing on:
- Migration guides (if version update)
- Breaking changes
- Best practices
- Configuration requirements
- Common pitfalls to avoid"
```

Example research topics:
- **Dependency updates**: Migration guides, breaking changes, deprecation notices
- **Tooling changes**: Setup guides, configuration options, CLI usage
- **Framework updates**: API changes, new patterns, compatibility requirements

#### 4b. Best Practices Research (perplexity-expert)

For implementation approach and industry best practices, use the perplexity-expert agent:

```
Task: perplexity-expert
Prompt: "Research best practices for [chore type] focusing on:
- Recommended approach for [specific task]
- Common mistakes to avoid
- Testing strategies
- Rollback procedures
- Industry standards for [technology]"
```

Example research topics:
- **Dependency updates**: Safe upgrade strategies, semver understanding, lock file management
- **Refactoring**: Code quality patterns, incremental migration strategies
- **Documentation**: Documentation standards, structure best practices
- **CI/CD changes**: Pipeline optimization, security best practices

### 5. Read Relevant Context Documentation

**Use the conditional documentation system** to load only the most relevant context:
```bash
# This will analyze the chore and load 3-7 relevant documentation files
slashCommand /conditional_docs chore-plan "[brief summary of chore or domain area]"
```

**Read each of the suggested documents** that are generated by /conditional_docs.

### 6. Solution Design Process

THINK HARD about the best approach to implement this chore:

1. **Synthesize research findings** from sub-agents
2. **Identify 2-3 potential implementation approaches**
   - Consider different strategies based on research
   - Evaluate risk vs speed trade-offs
   - Consider backward compatibility requirements
3. **Evaluate each approach**:
   - Pros and cons
   - Complexity (simple|moderate|complex)
   - Risk level (low|medium|high)
   - Alignment with research findings
   - Maintainability impact
4. **Select the recommended approach** with clear justification
5. **Design the detailed implementation strategy**:
   - What files need changes and why
   - What new files/configs are needed
   - What tests will validate success
   - Specific code patterns to follow
6. **Assess risks and create mitigation plan**
7. **Plan rollback strategy**

### 7. Create the Detailed Plan

Create the plan in `.ai/reports/chore-reports/YYYY-MM-DD/` using the **Plan Format** below:
- **Directory**: `.ai/reports/chore-reports/YYYY-MM-DD/` (use today's date)
- **Initial filename**: `pending-chore-plan-<slug>.md` where `<slug>` is a short kebab-case description
- IMPORTANT: Replace every <placeholder> with actual solution design
- Focus on **detailed implementation guidance**
- Include research findings from sub-agents
- Be thorough: provide enough detail that implementation is straightforward
- Include comprehensive testing and validation strategy
- Name the plan: `Chore Plan: <choreTitle>`
- **Example**: `.ai/reports/chore-reports/2025-11-27/pending-chore-plan-update-nextjs.md`

### 8. Update GitHub Issue

Use the **GitHub Issue Update** process to:
- Add detailed plan as comment to original chore issue
- Update labels to reflect planning complete
- Link research findings
