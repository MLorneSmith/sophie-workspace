## Initial Analysis

The root cause appears to be a mismatch between:

1. TypeScript compiling `.tsx` files to `.jsx` (due to `jsx: "preserve"` setting)
2. Package.json export map expecting `.js` files
3. The ESM import fixer script only processing certain files

The shared package builds successfully but outputs `.jsx` files while the package.json export configuration expects `.js` files, causing the module resolution to fail.

## Suggested Investigation Areas

1. Update package.json exports to match actual build output (.jsx instead of .js)
2. Configure TypeScript to output .js files for React components in the shared package
3. Update the fix-esm-imports.js script to handle events/index.jsx file
4. Consider changing jsx compilation setting in shared package tsconfig

- This issue may be related to recent changes for Issue #69 which involved TypeScript compilation fixes
