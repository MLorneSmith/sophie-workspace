## Summary

The project currently has both ESLint and Biome configured for linting, creating conflicting lint rules and preventing successful pre-commit hooks. The `lint:fix` command uses turbo to run ESLint across packages, while `lint-staged` uses Biome for linting, causing inconsistent behavior and failed commits.

## Reproduction Steps

1. Make any code changes to TypeScript/JavaScript files
2. Run `git commit` (triggers pre-commit hooks)
3. Observe that `biome lint --write --diagnostic-level=error` fails with linting errors
4. Run `pnpm lint:fix` - this uses ESLint instead of Biome
5. Note the conflicting behavior between the two linting systems

## Expected Behavior

- Single linting system (Biome) should be used for both formatting and linting
- Pre-commit hooks should pass consistently
- `lint:fix` command should use the same linting system as pre-commit hooks
- No conflicts between different linting configurations

## Actual Behavior

- ESLint is configured via turbo for the `lint:fix` command
- Biome is configured for pre-commit hooks via `lint-staged`
- Different rules and behaviors between the two systems
- Pre-commit hooks fail due to Biome linting errors that ESLint doesn't catch
- Inconsistent code quality enforcement

## Initial Analysis

The issue stems from the upstream MakerKit template using ESLint while the local project has been configured to use Biome. During the merge, both configurations were preserved, creating this conflict.

**Root Causes:**

1. **Script Inconsistency**: `lint:fix` uses turbo+ESLint while pre-commit uses Biome
2. **Dependency Bloat**: ESLint packages are still installed across the workspace
3. **Rule Conflicts**: Different linting rules between ESLint and Biome
4. **Tool Chain Confusion**: Developers don't know which tool to use for what

## Suggested Investigation Areas

1. **Remove ESLint Dependencies**: Clean up all `@kit/eslint-config` references
2. **Update Turbo Scripts**: Modify `lint` and `lint:fix` to use Biome instead of ESLint
3. **Workspace Cleanup**: Remove ESLint config files from packages
4. **Validation**: Ensure Biome can handle all current linting needs

## Recommended Solution

**Phase 1: Update Root Scripts**

```json
{
  "lint": "biome lint . && manypkg check",
```

- Remove `@kit/eslint-config` from all package.json files
