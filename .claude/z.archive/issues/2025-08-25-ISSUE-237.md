## Problem Summary

The `/test` command has multiple issues affecting delegation, visibility, and reliability. This consolidates findings from issues #235 and #236 with additional optimization opportunities identified through architectural analysis.

## Issues Identified

### 1. 🔴 Command Not Properly Delegating to Test Orchestrator
**Current Behavior:**
- `/test` command execution lacks proper Task tool delegation
- Direct execution bypasses orchestration system  
- No parallel E2E execution happening as designed

**Expected Behavior:**
- `/test` command should use Task tool to delegate to test-orchestrator agent
- test-orchestrator coordinates unit-test-agent and e2e-parallel-agent
- E2E tests run in 9 parallel shards for ~70% faster execution

### 2. 🔴 Nested Agent Execution Not Visible
**Problem:**
- Users can't see which agent is currently executing
- No progress updates during long-running tests
- TodoWrite not being used effectively for visibility

**Impact:**
- Poor user experience during 15-20 minute test runs
- Difficult to debug when tests hang or fail
- No way to track progress across parallel shards

### 3. ⚠️ Max-Failures Limiting Test Visibility  
**Issue:** All test:shard commands have `--max-failures=1` flag
**Location:** `apps/e2e/package.json` lines with test:shard[1-9]
**Impact:** E2E suite stops after first failure (only 25/85 tests ran in debug run)

### 4. 🐛 Statusline Component Not Updating
**Problem:**
- Test status in Claude Code statusline not receiving updates
- Status file path encoding issues
- Non-atomic file writes causing race conditions

### 5. 🟡 Bash Permission Friction
**Issue:** Repeated approval prompts for common test commands
**Impact:** Disrupts workflow and slows down test execution

## Root Cause Analysis

1. **Architecture is sound** - The orchestrator pattern with specialized agents is optimal
2. **Implementation gaps** - Task tool delegation not properly implemented
3. **Visibility gaps** - TodoWrite underutilized for progress tracking
4. **Configuration issues** - max-failures flag and permission settings need adjustment

## Recommended Fixes

### Priority 1: Fix Task Tool Delegation ✅

Update `/test` command to properly delegate:
```javascript
Task({
  subagent_type: "test-orchestrator",
  description: "Coordinate test execution", 
  prompt: `Execute comprehensive test suite with enhanced visibility.
    Options: ${JSON.stringify(options)}
    
    Requirements:
    1. Pre-flight infrastructure checks first
    2. Use TodoWrite for frequent progress updates
    3. Delegate unit tests to unit-test-agent via Task
    4. Delegate E2E tests to e2e-parallel-agent via Task
    5. Aggregate and report results
    
    Enable verbose output if DEBUG_TEST=true or --debug flag.`
})
```

### Priority 2: Enhance Visibility with TodoWrite 👁️

Implement hierarchical todo structure in agents:
- test-orchestrator: High-level phase tracking
- e2e-parallel-agent: Shard-level progress with real-time counts
- Frequent updates showing "Delegating to [agent-name]..." 
- Progress percentages and ETAs

### Priority 3: Remove max-failures Flag 📊

Update `apps/e2e/package.json`:
```diff
- "test:shard1": "playwright test tests/accessibility/accessibility-hybrid.spec.ts --max-failures=1",
+ "test:shard1": "playwright test tests/accessibility/accessibility-hybrid.spec.ts",
```
Apply to all 9 shard commands for full test visibility.

### Priority 4: Fix Statusline Updates 📈

Implement atomic status file writes:
```bash
update_status() {
    local status_file="/tmp/.claude_test_status_${GIT_ROOT//\//_}"
    local temp_file="${status_file}.tmp"
    
    echo "$1|$(date +%s)|$2|$3|$4" > "$temp_file"
    mv "$temp_file" "$status_file"  # Atomic operation
}
```

### Priority 5: Optimize Bash Permissions 🚀

Add to `.claude/settings.json`:
```json
{
  "allowedTools": [
    "Bash(pnpm test:*)",
    "Bash(npx supabase *)", 
    "Bash(pkill -f *)",
    "Bash(curl -s *)",
    "Bash(lsof -ti:*)"
  ]
}
```
