## Description
Development servers (web on port 3010, payload on port 3020) are being forcibly terminated with SIGTERM during E2E test execution, causing tests to hang indefinitely at 0% progress. This critical issue completely blocks E2E testing and prevents verification of other bug fixes.

## Impact
- **Severity**: Critical - Completely blocks E2E testing
- **Scope**: All E2E tests affected
- **Business Impact**: Cannot verify Issue #273 (authentication redirect fix) or any other E2E-dependent features
- **Development Impact**: E2E tests hang for 3+ minutes with no progress, requiring manual intervention

## Current Behavior
1. Unit tests execute successfully
2. Infrastructure checks pass
3. Development servers start correctly (web:3010, payload:3020)
4. E2E test execution begins
5. Both servers receive SIGTERM and exit with code 144
6. Tests hang at "0% completed" indefinitely
7. Manual kill required to stop hung test process

1. Incorrect timeout handling in test-controller.cjs
2. Process group management issues
3. Signal propagation problems with --quick flag
4. Race condition between server startup and test execution
