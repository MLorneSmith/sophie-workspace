## Initial Analysis

The issue stems from a monorepo configuration conflict where:

1. Two separate Vercel projects exist: `2025slideheroes-web` and `2025slideheroes-payload`
2. Each app has its own `vercel.json` in its directory
3. The root `vercel.json` was conflicting with app-specific configurations
4. Vercel is looking for Next.js in the root directory instead of `apps/web`

## Suggested Investigation Areas

1. Verify the Vercel project settings for `2025slideheroes-web` have the Root Directory set to `apps/web`
2. Confirm which Vercel project the `VERCEL_PROJECT_ID` secret is pointing to
3. Consider creating separate deployment workflows for web and payload apps
4. Review Vercel's monorepo documentation for best practices

## Additional Context

### Fixes Attempted:

1. ✅ Added missing `VERCEL_TOKEN` secret
2. ❌ Modified functions pattern in root vercel.json (still failed)
3. ❌ Removed functions configuration entirely (still failed)
4. ❌ Removed root vercel.json to avoid conflicts (still failed)

### Root Cause:

In a monorepo with multiple Vercel projects, the deployment needs to know which subdirectory contains the app. Without proper configuration, Vercel defaults to looking in the root directory where it cannot find Next.js.

### Solution:

The Vercel project settings need to specify the correct root directory (`apps/web` for the web project) either through:

- Vercel dashboard project settings
- Environment variables in the workflow
- Project-specific configuration
