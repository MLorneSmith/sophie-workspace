## Problem
E2E test shards are failing after exactly 2 seconds with 0 tests executed due to server startup coordination issues. The test controller is spawning separate processes for each shard, causing:
- Port conflicts (all shards try to bind to port 3000)
- Server startup race conditions 
- Bypassing Playwright's built-in server coordination

## Root Cause Analysis
Research identified that our test controller script bypasses Playwright's properly configured webServer coordination by running individual `pnpm test:shard{N}` commands that each attempt to start their own servers.

The Playwright config is correctly set up with:
- ✅ `reuseExistingServer: !process.env.CI`
- ✅ Proper webServer configuration for both Frontend/Backend
- ✅ `fullyParallel` enabled when `PLAYWRIGHT_PARALLEL=true`

## Solution: Use Playwright's Built-in Sharding

Replace custom shard commands with Playwright's native sharding:

### Current (Broken):
```json
{
  "test:shard1": "playwright test tests/accessibility/accessibility-hybrid.spec.ts",
  "test:shard2": "playwright test tests/authentication/auth.spec.ts"
}
```

### Fixed (Using Playwright Sharding):
```json
{
  "test:shard1": "PLAYWRIGHT_PARALLEL=true playwright test --shard=1/9",
  "test:shard2": "PLAYWRIGHT_PARALLEL=true playwright test --shard=2/9"
}
```
