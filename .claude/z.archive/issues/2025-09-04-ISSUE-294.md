## Description

Admin tests (Shard 2) are intentionally skipped when `TEST_SHARD_MODE=true` due to fundamental incompatibilities with parallel shard execution.

## Current Behavior

- **Location**: `apps/e2e/tests/admin/admin.spec.ts` lines 22-25
- **Impact**: 0/6 tests pass in shard mode (all tests are skipped)
- **Skip condition**: Tests are programmatically skipped when `process.env.TEST_SHARD_MODE === 'true'`

## Reasons for Skipping

1. **Hardcoded Admin Account**: Tests require a specific admin account (`michael@slideheroes.com`) that may not exist in the test database
2. **MFA Verification Issues**: Multi-factor authentication verification doesn't work reliably in shard execution mode
3. **Serial Execution Requirement**: Admin tests need serial execution mode which conflicts with shard parallelization strategy
4. **Database State Dependencies**: Admin operations require specific database state that's difficult to maintain across parallel shards

## Recommendations

### 1. Remove from Shard Configuration
Remove Admin tests from the shard configuration since they're always skipped anyway. This would:
- Reduce confusion about test coverage
- Prevent false sense of testing completeness
- Free up shard resources for actually executable tests

### 2. Separate Serial Execution
Run Admin tests separately in serial mode with proper environment setup:
```bash
TEST_SHARD_MODE=false pnpm test:admin --serial
```

### 3. Test-Specific Admin Account Seeding
Create a dedicated seeding mechanism for admin test accounts:
- Generate test-specific admin accounts on demand
- Use predictable credentials for test environments
- Clean up after test completion

### 4. MFA-Free Admin Test Mode
Implement an MFA-bypass mode specifically for E2E testing:
- Add environment variable `E2E_DISABLE_MFA=true`
- Only available in test environments
- Simplifies authentication flow for automated tests
