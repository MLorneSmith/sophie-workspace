## Summary

The E2E smoke test "CSS and JavaScript load properly" is failing because it cannot detect React on the page. The test evaluates the page to check for React markers (data-reactroot, #\_\_next, #root) but the check returns false, causing the test to fail after retries.

## Environment

- **Application Version**: 0.1.0
- **Environment**: development
- **Browser**: Chromium (Playwright)
- **Node Version**: 22.16.0
- **Next.js**: 15.3.4
- **Current Branch**: dev
- **Last Working**: Unknown
- **Related Issue**: Discovered while fixing #122

## Reproduction Steps

1. Start the development server: `pnpm dev`
2. Run the smoke tests: `pnpm playwright test tests/smoke/smoke.spec.ts`
3. Test "CSS and JavaScript load properly" fails with expected true, received false

## Expected Behavior

The test should detect that React is properly loaded on the page by finding one of:

- An element with `data-reactroot` attribute
- An element with id `#__next` (Next.js app container)
- An element with id `#root` (standard React app container)

## Test Code

The failing test is in `apps/e2e/tests/smoke/smoke.spec.ts`:

```typescript
test('CSS and JavaScript load properly', async ({ page }) => {
  await page.goto('/');

  // Check if React is loaded by looking for React-specific attributes
  const hasReact = await page.evaluate(() => {
    return (
      typeof window !== 'undefined' &&
      typeof document !== 'undefined' &&
      document.querySelector('[data-reactroot], #__next, #root') !== null
    );
  });
  expect(hasReact).toBe(true);
});
```

## Initial Analysis

1. **React Hydration**: The React root element might have a different selector in newer Next.js versions
2. **Timing Issue**: The test might be checking before React has fully hydrated
3. **App Structure Change**: The application might be using a different root element structure
4. **Test Selector**: The selector might need to be updated for the current Next.js/React version

## Suggested Investigation Areas

1. Check what root element Next.js 15 uses for the React app
2. Verify if the page is properly hydrating in development mode
3. Add a wait condition before checking for React elements
4. Update the selector to match the current app structure
5. Check if there are any console errors preventing React from loading
