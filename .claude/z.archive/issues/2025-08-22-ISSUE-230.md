## Problem Description

The dev.slideheroes.com domain is returning 522 errors (connection timeout) from Cloudflare, preventing access to the development environment and causing Dev Integration Tests to fail.

## Current Situation

### Symptoms
- **Error**: Cloudflare returns HTTP 522 (connection timeout) for all requests to https://dev.slideheroes.com
- **Impact**: Dev Integration Tests workflow gets stuck at "Wait for Deployment Ready"
- **Vercel Status**: Deployments complete successfully but create unique preview URLs for each deployment

### Investigation Results

1. **Vercel Deployments** (working):
   - Web app deploys to: `https://2025slideheroes-pk3y1ka1t-slideheroes.vercel.app`
   - Payload CMS deploys to: `https://2025slideheroes-payload-76xp9etux-slideheroes.vercel.app`
   - Both return HTTP 401 (unauthorized) when accessed directly - deployment protection is enabled

2. **Cloudflare** (failing):
   - dev.slideheroes.com configured in Cloudflare DNS
   - Returns 522 error - cannot connect to origin server
   - Likely pointing to wrong/outdated Vercel URL or missing authentication

3. **Workflow Configuration**:
   - Deploy to Dev workflow successfully deploys to Vercel
   - Dev Integration Tests expect dev.slideheroes.com to be accessible
   - Markdown-only changes don't trigger deployments (by design)

## Root Cause

The disconnect between Cloudflare and Vercel appears to be:
- Each Vercel deployment creates a unique preview URL
- Cloudflare is configured to point to a static URL that may no longer be valid
- Vercel deployment protection (401) prevents Cloudflare from accessing the origin

## Proposed Solutions

### Option 1: Add Custom Domain in Vercel (Recommended)
1. Add `dev.slideheroes.com` as a custom domain in Vercel project settings
2. Vercel will provide a CNAME target (e.g., `cname.vercel-dns.com`)
3. Update Cloudflare DNS to point to this CNAME
4. This ensures a stable endpoint regardless of deployment

### Option 2: Use Stable Production URL
1. Configure deployments to use a stable Vercel production URL
2. Update Cloudflare to proxy to this URL
3. Configure protection bypass headers if needed

### Option 3: Modify Deployment Workflow
1. Update workflow to use `vercel --prod --alias dev.slideheroes.com`
2. This creates a stable alias for the dev environment

### Option 4: Disable Deployment Protection
1. Disable Vercel deployment protection for dev environment
2. Or configure Cloudflare to send the `x-vercel-protection-bypass` header

## Temporary Workaround

Currently, the deployment workflow completes successfully, but the site remains inaccessible. To trigger a deployment after markdown-only changes, a non-markdown file must be modified.
