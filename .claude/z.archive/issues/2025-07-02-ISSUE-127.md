## Summary

When running `git commit`, lint-staged processes are being forcefully terminated with SIGKILL errors during concurrent task execution. This occurs even though individual commands run successfully when executed manually. The issue appears to be related to resource constraints or process management when multiple lint-staged tasks run in parallel.

## Environment

- **Application Version**: 0.1.0
- **Environment**: development
- **Node Version**: 22.16.0
- **pnpm Version**: 9.12.0
- **lint-staged Version**: 16.1.0
- **Last Working**: Unknown

## Reproduction Steps

1. Make changes to multiple file types (e.g., both package.json and apps/payload files)
2. Stage the files with `git add`
3. Attempt to commit with `git commit -m "message"`
4. Multiple lint-staged processes start in parallel
5. Several processes are terminated with SIGKILL

## Expected Behavior

All lint-staged tasks should complete successfully without being forcefully terminated.

## Actual Behavior

Multiple processes receive SIGKILL during execution:

- TruffleHog scan process
- Biome lint process for Payload
- TypeScript checking process

Error output shows:

```
[FAILED] bash ./scripts/trufflehog-scan.sh [SIGKILL]
[FAILED] bash -c 'cd apps/payload && pnpm lint' [SIGKILL]
[FAILED] bash -c 'echo "🔍 Type checking $0..."; tsc --noEmit --skipLibCheck --pretty "$0" 2>/dev/null || echo "⚠️ Type check failed for $0"' [SIGKILL]
```

## Initial Analysis

The SIGKILL errors suggest:

1. **Resource exhaustion**: System may be killing processes due to memory/CPU limits
2. **Process group termination**: Parent process manager might be terminating child processes
3. **Concurrency overload**: Default lint-staged concurrency might be too high

The issue is distinct from #126 (spawn cd ENOENT) which has been resolved.

## Suggested Investigation Areas

1. **Limit lint-staged concurrency**: Configure lint-staged to run fewer tasks in parallel
2. **Check system resource limits**: ulimit settings, Docker/WSL memory limits
3. **Sequential execution**: Test with sequential task execution to confirm it's concurrency-related
4. **Process monitoring**: Use system tools to monitor resource usage during commit
