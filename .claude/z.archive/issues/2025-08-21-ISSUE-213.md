## Description
The E2E accessibility tests are failing due to WCAG 2.1 AA color contrast violations on the authentication pages. This prevents the test suite from completing successfully.

## Error Details
The accessibility test for authentication pages (`/auth/sign-in`) is failing with the following violations:

### Color Contrast Violations
1. **Google Sign-in Button Text**
   - Current contrast ratio: 1.69:1
   - Required contrast ratio: 4.5:1
   - Foreground color: #c7c7c7
   - Background color: #ffffff
   - Element: `<span class="font-medium" style="color:hsl(0 0% 15%)">Sign in with Google</span>`

2. **Sign-up Link**
   - Current contrast ratio: 1.76:1
   - Required contrast ratio: 4.5:1
   - Foreground color: #c1c3c8
   - Background color: #ffffff
   - Element: `<a class="...text-foreground" href="/auth/sign-up">`
