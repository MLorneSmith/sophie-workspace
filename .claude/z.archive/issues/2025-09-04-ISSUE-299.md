After successfully creating 19 new E2E tests to fix issue #298, the tests work perfectly when run directly with Playwright but fail when executed through the test controller (test-controller.cjs).

- `apps/e2e/tests/account/account-simple.spec.ts` - 9 tests

## Likely Root Causes
1. **Working Directory Context**: Controller may be executing tests from wrong directory
2. **Environment Variables**: Different env vars between direct run and controller (e.g., PLAYWRIGHT_BASE_URL)
3. **Server Startup Timing**: Dev servers might not be fully ready when controller runs tests
4. **Shard Execution**: Controller's parallel shard system may have race conditions
5. **Path Resolution**: Relative paths in tests might resolve differently in controller context
