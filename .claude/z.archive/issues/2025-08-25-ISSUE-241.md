## Initial Analysis

1. **Root Cause**: The PlanPicker component is conditionally rendered based on `!hasBillingData`. During E2E tests, the test users likely already have billing data (subscription or order), causing the PlanPicker to not render.

2. **Not a Test Selector Issue**: The `data-test-plan` attributes are correctly present in the code, matching the upstream MakerKit implementation exactly.

3. **Test Data Issue**: The problem appears to be with test data setup - test users should start with no billing data to allow the PlanPicker to render.

## Suggested Investigation Areas

1. **Check test user setup**: Verify that test users start with no subscriptions or orders
2. **Database state**: Check if Supabase E2E instance has lingering billing data for test users
3. **Test isolation**: Ensure each test properly cleans up billing data after completion
4. **Add debug logging**: Log `hasBillingData` value during tests to confirm the hypothesis
