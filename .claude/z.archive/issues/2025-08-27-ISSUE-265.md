## Description
The current log-issue agent (`.claude/agents/log-issue.md`) needs to be enhanced to check for existing open issues before creating new ones to prevent duplicate issue creation.

## Current Behavior
- The agent currently creates new issues without checking if similar issues already exist
- This can lead to duplicate issues being created for the same problem/topic
- Line 38 mentions "Verify the issue hasn't been previously logged" but this is not implemented

## Proposed Enhancement

### 1. Pre-creation Checks
Before creating a new issue, the agent should:
- Search for existing open issues with similar titles or keywords
- Compare the new issue's description with existing issues
- If a potential duplicate is found, either:
  - a. Add a comment to the existing issue with new information
  - b. Ask the user if they want to create a new issue anyway
  - c. Link the issues as related

### 2. Implementation Suggestions
- Use GitHub API search to find similar issues
- Implement fuzzy matching for title/description comparison
- Add a similarity threshold (e.g., 80% match = likely duplicate)
- Store a cache of recent issues to improve performance

## Expected Benefits
- Reduces issue clutter and duplicates
- Improves issue tracking efficiency
- Better consolidation of related problems
- Saves team time in issue triage
