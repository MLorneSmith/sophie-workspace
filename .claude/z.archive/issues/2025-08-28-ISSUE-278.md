## Context

While debugging Issue #273 (authentication redirect failures), we discovered that the root cause was missing test user data in the database. The E2E tests were failing because test users (`test1@slideheroes.com`, `test2@slideheroes.com`) weren't properly seeded, causing authentication to fail with "invalid credentials" errors.

## Current State

- Test users are seeded via `apps/e2e/supabase/seeds/01-e2e-test-data.sql` when running `npx supabase db reset`
- However, this manual process is not integrated into our automated test workflows
- Tests can fail unexpectedly if developers run tests without proper database seeding
- The `.claude/scripts/test-controller.cjs` script doesn't validate test data prerequisites

## Proposed Enhancement

Enhance the test infrastructure to automatically check for and seed required test users before running E2E tests.
