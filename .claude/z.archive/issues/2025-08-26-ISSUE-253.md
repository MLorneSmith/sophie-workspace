### Root Cause Analysis

#### Primary Issue
The Playwright test runner is unable to find any tests marked with the `@integration` tag when executing the command:
```bash
pnpm --filter web-e2e test:integration
```

This command translates to:
```bash
playwright test --grep @integration --project=chromium --retries=1
```

#### Evidence
1. **Workflow Configuration** (`.github/workflows/dev-integration-tests.yml`, line 270):
   - The workflow runs `pnpm --filter web-e2e test:integration`
   - This command searches for tests tagged with `@integration`

2. **Test Files Analysis**:
   - 6 test files contain `@integration` tags:
     - `/tests/invitations/invitations.spec.ts`
     - `/tests/team-billing/team-billing.spec.ts` 
     - `/tests/user-billing/user-billing.spec.ts`
     - `/tests/authentication/auth.spec.ts`
     - `/tests/authentication/password-reset.spec.ts`
     - `/tests/team-accounts/team-accounts.spec.ts`

3. **Configuration Issue** (`apps/e2e/playwright.config.ts`):
   - Lines 31-45: Test files are being ignored based on environment variables
   - `ENABLE_BILLING_TESTS` defaults to `false` (line 17)
   - `ENABLE_TEAM_ACCOUNT_TESTS` defaults to `true` (lines 18-19)
   - When billing tests are disabled, `*-billing.spec.ts` files are ignored (line 31)
   - This excludes 2 of the 6 integration test files

4. **Missing Environment Variables**:
   - The CI workflow doesn't set `ENABLE_BILLING_TESTS=true`
   - This causes `team-billing.spec.ts` and `user-billing.spec.ts` to be excluded
   - If all remaining integration tests are somehow skipped or fail to match, Playwright exits with no tests to run
