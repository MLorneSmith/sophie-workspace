## Summary

The E2E integration tests are failing in CI due to an ESM module import error with chrome-launcher v1.2.0. The package has been updated to ESM-only, but our TypeScript configuration is transpiling imports to CommonJS `require()` calls, which cannot load ESM modules. This is blocking the CI/CD pipeline from automatically promoting code from dev to staging.

## Initial Analysis

The chrome-launcher package v1.2.0 has been converted to ESM-only. Our TypeScript/build configuration is compiling the import statements to CommonJS `require()` calls, which cannot load ESM modules. This is a breaking change in the chrome-launcher package that requires either:
1. Updating our build configuration to handle ESM modules
2. Using dynamic imports
3. Downgrading chrome-launcher to a CommonJS-compatible version

## Suggested Investigation Areas

1. **Quick Fix Options**:
   - Downgrade chrome-launcher to v1.1.x (last CommonJS version)
   - Convert to dynamic import: `const { launch } = await import("chrome-launcher")`

2. **Proper Fix Options**:
   - Update TypeScript configuration to output ESM
   - Configure Playwright test runner to handle ESM modules
   - Update Node.js loader to support ESM in test environment

3. **Workaround**:
   - Skip affected accessibility tests temporarily
   - Use alternative Chrome launcher implementation
