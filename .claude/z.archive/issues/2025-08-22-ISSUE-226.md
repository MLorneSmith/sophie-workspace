1. Create a test data seeding script that runs before admin tests
2. Modify tests to dynamically create admin users instead of using hardcoded credentials
3. Add test fixtures with predefined users

```typescript
// Instead of hardcoded credentials
await auth.signIn({
  email: "test1@slideheroes.com",
```

### Fix 2: Onboarding Flow Consistency

Options:
1. Update tests to handle both onboarding and direct-to-home scenarios
2. Create separate test suites for with/without autoconfirm
3. Make onboarding detection more flexible

Example approach:
```typescript
// Instead of expecting onboarding
await page.waitForURL("**/onboarding");

// Handle both scenarios
await page.waitForURL((url) => 
  url.href.includes("/onboarding") || url.href.includes("/home"),
  { timeout: 10000 }
);

const currentUrl = page.url();
if (currentUrl.includes("/onboarding")) {
  await onboarding.completeOnboardingSimple();
}
```
