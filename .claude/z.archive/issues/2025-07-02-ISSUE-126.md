## Summary

The pre-commit hook fails when trying to commit files in the `apps/payload` directory because lint-staged is attempting to execute `cd` as a spawnable command, which fails with a `spawn cd ENOENT` error. This prevents normal commits of Payload-related files and requires using `--no-verify` to bypass the pre-commit hook.

1. Make changes to any file in `apps/payload/` (e.g., `payload-types.ts` or seed data JSON files)
2. Stage the files with `git add`
3. Attempt to commit with `git commit -m "message"`
4. Pre-commit hook fails with spawn error

## Expected Behavior

The lint-staged command should successfully:

1. Change directory to `apps/payload`
2. Run `pnpm lint --fix` on the staged files

- Command tries to spawn `cd` as a command
- Error: `spawn cd ENOENT`

## Initial Analysis

The issue occurs because lint-staged is not executing the command in a shell context. When it sees `cd apps/payload && pnpm lint --fix`, it tries to spawn `cd` as an executable, which doesn't exist as a standalone command. The `&&` is being passed as a literal argument instead of a shell operator.

## Suggested Investigation Areas

1. **Wrap command in shell execution**: Use `bash -c` to ensure the command runs in a shell context
2. **Use lint-staged functions**: Replace string command with a function that properly handles directory changes
3. **Use relative paths**: Configure the lint command to accept full paths and run from project root
4. **Alternative approach**: Create a script in `apps/payload/package.json` that lint-staged can call directly

## Proposed Solutions

### Option 1: Use bash -c wrapper

```json
"apps/payload/**/*.{js,jsx,ts,tsx,json}": [
  "bash -c 'cd apps/payload && pnpm lint --fix'"
]
```

### Option 2: Use lint-staged function syntax

```javascript
module.exports = {
  'apps/payload/**/*.{js,jsx,ts,tsx,json}': async (files) => {
    const cwd = path.join(process.cwd(), 'apps/payload');
    return `pnpm --dir ${cwd} lint --fix ${files.join(' ')}`;
  },
};
```

### Option 3: Run from root with proper paths

```json
"apps/payload/**/*.{js,jsx,ts,tsx,json}": [
  "pnpm --filter payload lint --fix"
]
```

## Additional Context

- This issue was discovered while creating separate commits for a large changeset
- Workaround was to use `git commit --no-verify` to bypass the pre-commit hook
- Other lint-staged commands (like those for packages) use `bash -c` wrapper which works correctly
