## Initial Analysis

The issue appears to be that NewRelic (a server-side monitoring tool) is being imported in a way that Next.js/webpack is attempting to bundle it for client-side use. The newrelic package contains non-JavaScript files and uses Node.js built-in modules that cannot be parsed or resolved in a browser context.

This suggests either:

1. The newrelic module is being imported in a client component/module
2. The Next.js configuration needs to exclude newrelic from client-side bundling
3. The instrumentation setup needs to be properly configured for server-side only

## Suggested Investigation Areas

1. Check if `instrumentation.ts` is properly configured for server-side only execution
2. Review the import chain to ensure newrelic is not being imported in any client components
3. Consider adding webpack configuration to exclude newrelic from client bundling
4. Verify Next.js experimental instrumentation feature is properly configured

## Additional Context

- Using Next.js 15.3.4 with experimental features
- Monorepo setup with pnpm workspaces
- NewRelic is intended for server-side APM monitoring only
