## Summary

Convert raw seed data files (mdoc, yaml, html, ts) located in `apps/payload/src/seed-data-raw/` into JSON format compatible with Payload CMS's Local API seeding system. This involves parsing various file formats, extracting media references that already exist in Cloudflare R2 storage, and creating properly structured JSON files that respect collection dependencies and relationships.

## Success Criteria

- [ ] All mdoc files converted to JSON with Lexical-formatted content
- [ ] All yaml survey files converted to JSON format
- [ ] Media and download references extracted and mapped to R2 URLs
- [ ] Course structure generated from lesson data
- [ ] All cross-collection relationships properly referenced using `{ref:collection:identifier}` format
- [ ] Generated JSON validates against Payload TypeScript types
- [ ] Seed data can be successfully imported via Payload Local API

## Technical Requirements

### Dependencies

- Payload CMS seed protocol design (see `.claude/scratch/payload-seed-protocol-design.md`)
- gray-matter (for frontmatter parsing)
- js-yaml (for YAML parsing)
- Payload's Lexical editor format
- Cloudflare R2 storage configuration

### Constraints

- No user data will be seeded (as specified)
- Media/download files already exist in R2 - only database records needed
- Must respect collection dependency order for seeding
- Must preserve all metadata from original files
