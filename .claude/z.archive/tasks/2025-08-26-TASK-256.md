## Problem Statement
The /test command and agents performed moderately well but showed critical issues affecting test reliability and coverage:
- Only 60/245 expected unit tests ran (likely cached partial results)
- E2E tests have ~50% failure rate due to missing selectors and aggressive timeouts
- No automatic recovery or retry mechanisms for failures
- Critical business features are failing tests

## Current State
- **Execution time**: ~3.5 minutes (good)
- **Unit tests**: 60/60 passed but incomplete coverage
- **E2E tests**: ~45/90 passed with failures in admin, auth, invitations, smoke tests, and team/billing
- **Critical business features are failing**

## Proposed Improvements

### 1. Unit Test Agent Enhancements
- Verify all workspaces are tested (not just cached)
- Force fresh runs with `TURBO_FORCE=true` when needed
- Add per-workspace timeout monitoring
- Implement retry logic for flaky tests
- Report which workspaces were skipped

### 2. E2E Parallel Agent Improvements
- Increase element detection timeout from 10s to 20s
- Add automatic retry for timeout failures (max 2 attempts)
- Capture screenshots on failure for debugging
- Pre-validate critical selectors exist before running
- Add better error classification (env issues vs actual failures)

### 3. Test Orchestrator Enhancements
- Implement intelligent failure classification system
- Add auto-fix attempts for common issues:
  * Missing selectors → suggest selector additions
  * Timeouts → retry with increased wait
  * Port conflicts → auto-cleanup and retry
- Generate actionable fix commands for failures

### 4. Infrastructure Robustness
- Verify database connection and migrations are current
- Check Next.js build validity: `pnpm build`
- Validate environment variables: `test -f .env.test && source .env.test`
- Clean zombie processes: `pkill -f "playwright|vitest"`
- Add port cleanup between test runs

### 5. Performance Optimizations
- Implement git-based test selection (only run affected tests)
- Smart test sharding based on historical timing data
- Cache test dependencies between runs
- Parallelize unit and E2E infrastructure setup

### 6. Script-Level Improvements
Add to `test-controller.js`:
- Retry mechanism with exponential backoff
- Configurable timeouts per test type
- Better error messages with context
- Progress tracking improvements

The test infrastructure uses:
- `test-runner.sh`: Bash wrapper for argument parsing
- `test-controller.js`: Node.js orchestrator
- Parallel E2E execution across 9 shards
- Turbo for workspace-level parallelization
