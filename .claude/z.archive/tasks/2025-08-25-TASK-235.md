## Overview

Based on architectural analysis, the `/test` command's orchestrator pattern is optimal but needs visibility and reliability improvements. The current architecture correctly handles 129+ tests across 9 parallel E2E shards, but subagent execution isn't visible in Claude Code UI.

## Background

- **Current Architecture**: `/test` → `test-orchestrator` → [`unit-test-agent`, `e2e-parallel-agent`]
- **Issue**: Nested agents operate in isolated contexts, reducing visibility
- **Finding**: Architecture is NOT redundant - it's a best-practice implementation for complex workflows

## Implementation Tasks

### 1. 🔴 Enhance Subagent Visibility (High Priority)

- [ ] Add verbose TodoWrite updates in orchestrator showing delegations
- [ ] Include "Delegating to [agent-name]..." status messages  
- [ ] Surface key metrics from subagents in orchestrator output
- [ ] Add `--debug` flag for detailed execution trace
- [ ] Log subagent activity to temp files for debugging

### 2. 🟡 Add Direct Agent Access Commands (Medium Priority)

- [ ] Implement `/test-unit` for direct unit-test-agent invocation
- [ ] Implement `/test-e2e` for direct e2e-parallel-agent invocation

- [ ] Implement `/test-infra` for infrastructure validation only

## Technical Implementation Details

### Enhanced Orchestrator Prompt
```yaml
VISIBILITY_REQUIREMENTS: |
  - Before each delegation: Update TodoWrite with "Delegating to [agent]"
  - After each delegation: Report key metrics in main output
  - Use descriptive status messages throughout
  - Include timing for each phase
```

### Debug Mode Implementation
```yaml
debug:
  description: "Show detailed execution trace"
  implementation: |
    - Set DEBUG_TEST=true environment variable
    - Agents check this and increase verbosity
    - Log all operations to /tmp/test-debug.log
```

### Flaky Test Handling
```yaml
flaky_test_retry:
  detection: "Test failed with timeout or network error"
  action: "Retry test in isolation"
  max_attempts: 2
  isolation: "Fresh browser context"
```
