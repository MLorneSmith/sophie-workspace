## Description

We need a comprehensive logging system to track and debug Claude Code subagents during execution. This will help us understand agent behavior, diagnose failures, and improve orchestration efficiency.

## Problem Statement

Currently, debugging subagent execution is challenging due to:
- Lack of visibility into agent decision-making processes
- Difficulty tracking parent-child relationships in orchestration chains
- No systematic way to replay or analyze agent failures
- Missing performance metrics for optimization

## Proposed Solution

### Core Components

#### 1. Centralized Logging Service (AgentLogger)
- **Format**: JSON-structured logs with ISO timestamps
- **Log Levels**: `DEBUG`, `INFO`, `WARN`, `ERROR`
- **Context Injection**: Automatic injection of:
  - Agent ID
  - Parent Agent ID
  - Session ID
  - Execution context
- **Rotation Policy**: Daily rotation with 30-day retention

#### 2. Agent Execution Wrapper
- Track complete agent lifecycle:
  - Initialization
  - Tool invocations
  - Decision points
  - Completion/failure
- Capture comprehensive data:
  - Input parameters
  - Output results
  - Error messages with stack traces
  - Execution duration
- Parent-child relationship mapping for orchestration chains

#### 3. Storage Strategy
- **Primary Storage**: SQLite database
  - Structured querying capability
  - Efficient filtering and aggregation
  - Schema versioning support
- **Secondary Storage**: JSON files
  - Backup/portability
  - Human-readable format
  - Directory structure: `.claude/logs/agents/[date]/[session-id]/`

#### 4. Key Features
- **Tool Call Tracking**: Complete parameter and result logging
- **Timeline Visualization**: Execution flow with timestamps
- **Error Capture**: Full stack traces and context
- **Performance Metrics**:
  - Execution duration
  - Token usage (if applicable)
  - Memory consumption
  - Tool call frequency
- **Relationship Mapping**: Parent-child agent hierarchy

#### 5. Debug Utilities
- **Log Viewer CLI**:
  - Filter by agent ID, time range, severity
  - Real-time tail functionality
  - Export capabilities
- **Execution Replay**:
  - Reconstruct agent decision flow
  - Step-through debugging
- **Performance Analyzer**:
  - Identify bottlenecks
  - Generate performance reports
- **Error Aggregator**:
  - Pattern recognition
  - Common failure identification

## Technical Specifications

### Log Schema
```json
{
  "timestamp": "ISO 8601",
  "level": "DEBUG|INFO|WARN|ERROR",
  "agent_id": "unique_identifier",
  "parent_id": "parent_agent_id",
  "session_id": "session_identifier",
  "event_type": "start|tool_call|decision|complete|error",
  "message": "human_readable_message",
  "data": {
    "tool": "tool_name",
    "parameters": {},
    "result": {},
    "duration_ms": 0,
    "error": {}
  }
}
```

### SQLite Schema
```sql
CREATE TABLE agent_logs (
  id INTEGER PRIMARY KEY,
  timestamp DATETIME,
  level TEXT,
  agent_id TEXT,
  parent_id TEXT,
  session_id TEXT,
  event_type TEXT,
  message TEXT,
  data JSON,
  INDEX idx_agent_id (agent_id),
  INDEX idx_session_id (session_id),
  INDEX idx_timestamp (timestamp)
);
```
