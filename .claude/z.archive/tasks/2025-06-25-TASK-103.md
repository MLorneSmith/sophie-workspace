## Intelligent Test Selection Logic

### Priority Matrix Factors
1. **Test Coverage Gaps**: Areas with no tests get highest priority
2. **CI/CD Stage Requirements**: Tests needed for next deployment stage
3. **Business Impact**: Revenue-critical functionality prioritized
4. **Test Dependencies**: Unit tests before integration, integration before E2E
5. **Recent Changes**: Modified code areas need corresponding tests

### Test Type Selection Algorithm
```
1. Analyze current test coverage across all types
2. Identify next CI/CD pipeline requirements
3. Score untested functionality by business impact
4. Apply test dependency rules (unit → integration → E2E)
5. Select highest-priority test type and specific target
6. Generate appropriate test using type-specific templates
```

### Override Options
- `--type=unit|integration|e2e|accessibility|performance` - Force specific test type
- `--target=<file_path>` - Force specific file testing
- `--stage=pr|dev|staging|production` - Focus on specific CI/CD stage requirements

### Integration Tests
