## Background

The current `/test` command implementation (TASK-257) has revealed critical issues that require a strategic refactor approach:

1. **Agent logging initialization failures** - Complex custom SQLite/JSON logging system fails to initialize properly
2. **E2E timeout issues** - 2-minute timeout limits cause test failures when E2E shards run in single long-running Bash sessions
3. **Over-engineered logging complexity** - TypeScript-based agent logging conflicts with Claude Code's native capabilities

This issue proposes a **pragmatic simplification approach** that leverages Claude Code's built-in OpenTelemetry support rather than fighting against it.

## Problem Analysis

### Current Issues (from TASK-257 implementation)
- Custom agent logging (`agent-logger.ts`, `agent-task-logger.ts`, `agent-execution-wrapper.ts`) fails during initialization
- Complex TypeScript initialization conflicts with Claude Code's execution model  
- E2E test shards timeout after 2 minutes due to running as single long Bash session
- 66 E2E tests across 9 shards cannot complete within timeout constraints

### Root Cause
We're implementing a custom logging system that duplicates and conflicts with Claude Code's native OpenTelemetry + hooks approach.

## Proposed Solution

### 1. Agent Logging Migration (Simplification)

**Remove Complex Custom System:**
- Delete `.claude/utils/agent-logger.ts`
- Delete `.claude/utils/agent-task-logger.ts`  
- Delete `.claude/utils/agent-execution-wrapper.ts`
- Remove all TypeScript-based logging initialization

**Replace with Native OpenTelemetry:**
- Use environment variables for OpenTelemetry configuration
- Create simple `.claude/hooks/post-tool-use.py` for basic task delegation logging
- Leverage Claude Code's built-in tracing and logging capabilities

### 2. E2E Timeout Fix Implementation

**Current Problem:**
```bash
# This times out after 2 minutes
for shard in {1..9}; do
  pnpm test:shard$shard &
done
wait  # <- Timeout occurs here
```

**Proposed Solution:**
```bash
# Each shard runs as individual Bash command with 10-minute timeout
Bash: pnpm test:shard1 (run_in_background: true, timeout: 600000)
Bash: pnpm test:shard2 (run_in_background: true, timeout: 600000)
# ... etc
# Use BashOutput tool to monitor completion
```

  ```python
  #!/usr/bin/env python3
  import json
  import sys
  from datetime import datetime
  
  # Simple task delegation logging
  def log_task_delegation(tool_name, tool_args):
      timestamp = datetime.now().isoformat()
      log_entry = {
          "timestamp": timestamp,
          "tool": tool_name,
          "args": tool_args,
          "type": "task_delegation"
      }
      print(json.dumps(log_entry))
  
  if __name__ == "__main__":
      tool_data = json.loads(sys.argv[1])
      log_task_delegation(tool_data.get("name"), tool_data.get("args"))
  ```
