## Implementation Pattern

### Before (hardcoded colors):
```tsx
<div className="bg-white text-gray-900 border-gray-200">
  <h2 className="text-blue-600">Title</h2>
  <button className="bg-blue-500 text-white hover:bg-blue-600">
    Click
  </button>
</div>
```

### After (theme-aware):
```tsx
<div className="bg-card text-card-foreground border-border">
  <h2 className="text-primary">Title</h2>
  <button className="bg-primary text-primary-foreground hover:bg-primary/90">
    Click
  </button>
</div>
```

## Component Update Examples

### 1. Card Component
```tsx
// Before
<div className="bg-white shadow-md rounded-lg p-6">
  <h3 className="text-gray-900 font-semibold">{title}</h3>
  <p className="text-gray-600">{description}</p>
</div>

// After
<div className="bg-card shadow-md dark:shadow-xl rounded-lg p-6 border border-border">
  <h3 className="text-card-foreground font-semibold">{title}</h3>
  <p className="text-muted-foreground">{description}</p>
</div>
```

### 2. Input Component
```tsx
// Before
<input
  className="border border-gray-300 rounded px-3 py-2 
             focus:border-blue-500 focus:outline-none"
  {...props}
/>

// After
<input
  className="border border-input rounded px-3 py-2 
             bg-background text-foreground
             focus:border-primary focus:outline-none 
             focus:ring-2 focus:ring-ring"
  {...props}
/>
```

### 3. Button Component
```tsx
// Before
<button className="bg-blue-500 text-white px-4 py-2 rounded 
                   hover:bg-blue-600 disabled:bg-gray-300">
  {children}
</button>

// After
<button className="bg-primary text-primary-foreground px-4 py-2 rounded 
                   hover:bg-primary/90 disabled:bg-muted 
                   disabled:text-muted-foreground">
  {children}
</button>
```

## Testing Strategy

### Visual Testing Checklist
For each component, verify:
1. **Light Mode**
   - Proper background color
   - Text readability
   - Border visibility
   - Hover/focus states
   - Disabled states

2. **Dark Mode**
   - Proper background color
   - Text readability
   - Border visibility
   - Hover/focus states
   - Disabled states

3. **Transition**
   - Smooth color transitions
   - No layout shifts
   - No flashing

### Automated Testing
```typescript
// Example test for theme-aware component
describe('Component Dark Mode', () => {
  it('applies correct classes in dark mode', () => {
    document.documentElement.classList.add('dark');
    const { container } = render(<Component />);
    expect(container.firstChild).toHaveClass('dark:bg-gray-800');
  });
});
```

## Common Pitfalls to Avoid

1. **Don't use Tailwind's dark: prefix everywhere**
   - Use CSS variables instead for consistency

2. **Don't forget hover and focus states**
   - These need dark mode variants too

3. **Watch for opacity utilities**
   - `bg-black/10` might need different values in dark mode

4. **Test with real content**
   - Long text, different states, edge cases
