# Task: Build reference resolution engine

## Description
Implement the reference resolution engine that parses `{ref:collection:identifier}` patterns, maintains an in-memory UUID cache, and recursively resolves references throughout data structures. This is a critical component for establishing relationships between records.

## Acceptance Criteria
- [ ] Reference parser implemented with regex pattern matching
- [ ] In-memory cache implemented with Map-based storage
- [ ] Recursive resolution algorithm working for all data structures
- [ ] All reference patterns supported (simple, array, nested, path-based)
- [ ] Cache registration and lookup working correctly
- [ ] Validation for unresolved references
- [ ] Unit tests passing with >95% coverage

## Technical Details
### Implementation Approach
1. Create `reference-resolver.ts` with:
   - `register(collection, identifier, uuid)` - Add to cache
   - `resolve(record)` - Recursively resolve all references
   - `validate(record)` - Check all references can be resolved
   - `getCache()` - Get current cache state
2. Implement deep resolution algorithm:
   - Handle strings with reference patterns
   - Handle arrays of references
   - Handle nested objects recursively
   - Handle mixed content
3. Support all reference patterns:
   - Simple: `"{ref:courses:ddm}"` → UUID
   - Array: `["{ref:downloads:t1}", "{ref:downloads:t2}"]` → [UUID1, UUID2]
   - Nested: `{"author": {"id": "{ref:users:admin}"}}` → Resolved object
   - Path-based: `"{ref:media:/cms/images/thumb.png}"` → Media UUID

### Files to Create
- `apps/payload/src/seed/seed-engine/resolvers/reference-resolver.ts` - Resolution engine
- `apps/payload/src/seed/seed-engine/resolvers/reference-resolver.test.ts` - Comprehensive tests

### Algorithm Complexity
- Time: O(n) where n = number of fields in object
- Space: O(d) where d = depth of nesting
- Cache lookup: O(1) with Map

## Risk Assessment
### Potential Issues
- Circular references may cause infinite loops
- Performance may degrade with deeply nested structures
- Reference patterns may be ambiguous
- Cache may grow too large

### Mitigation Strategies
- Add cycle detection
- Limit recursion depth
- Use clear, unambiguous patterns
- Cache is ephemeral (cleared after each run)
- Comprehensive unit tests for edge cases
