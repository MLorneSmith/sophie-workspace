# Task: Create directory structure and TypeScript types

## Description
Set up the foundational directory structure for the Payload seeding engine and define all core TypeScript types and interfaces. This establishes the type-safe foundation for the entire seeding system.

## Acceptance Criteria
- [ ] Directory structure created: `apps/payload/src/seed/seed-engine/`
- [ ] All subdirectories created: `core/`, `loaders/`, `resolvers/`, `processors/`, `validators/`, `utils/`
- [ ] `types.ts` created with complete type definitions
- [ ] `config.ts` created with constants and seed order configuration
- [ ] All types are exported and properly documented with JSDoc
- [ ] No TypeScript errors in type definitions
- [ ] Type definitions cover all identified use cases

## Technical Details
### Implementation Approach
1. Create the complete directory structure following the plan
2. Define TypeScript interfaces for all core entities:
   - `SeedRecord` - Base type for seed data records
   - `CollectionConfig` - Collection configuration type
   - `ReferenceCache` - Reference resolution cache type
   - `ProcessorResult` - Result type for processing operations
   - `SeedOptions` - CLI options and configuration
   - `ProgressReport` - Progress tracking data
3. Define seed order constants and configuration
4. Add comprehensive JSDoc comments for all types

### Files to Create
- `apps/payload/src/seed/seed-engine/types.ts` - All TypeScript type definitions
- `apps/payload/src/seed/seed-engine/config.ts` - Configuration constants and seed order
- Directory structure for all modules
