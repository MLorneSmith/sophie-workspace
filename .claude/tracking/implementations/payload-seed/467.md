# Task: Add data validators

## Description
Implement pre-seeding validators that check data quality before attempting to seed. This includes dependency validation, reference validation, and post-seed verification to ensure data integrity.

## Acceptance Criteria
- [ ] Dependency validator checks collection order
- [ ] Reference validator checks all references can be resolved
- [ ] Required field validator checks for missing data
- [ ] Post-seed validator checks record counts and relationships
- [ ] Clear error messages for all validation failures
- [ ] Unit tests passing with >90% coverage

## Technical Details
### Implementation Approach
1. Create `dependency-validator.ts`:
   - Verify collection dependencies in SEED_ORDER
   - Check for circular dependencies
   - Validate all referenced collections exist
2. Create `data-validator.ts`:
   - Validate all `{ref:}` patterns are syntactically correct
   - Check all referenced collections exist in Payload config
   - Verify required fields present in all records
   - Check field types for critical fields
3. Create `post-seed-validator.ts`:
   - Verify record counts match expectations
   - Sample relationship integrity checks
   - Verify Lexical content is parseable
   - Check for orphaned relationships

### Files to Create
- `apps/payload/src/seed/seed-engine/validators/dependency-validator.ts` - Dependency checks
- `apps/payload/src/seed/seed-engine/validators/data-validator.ts` - Data quality checks
- `apps/payload/src/seed/seed-engine/validators/post-seed-validator.ts` - Post-seed verification
- `apps/payload/src/seed/seed-engine/validators/dependency-validator.test.ts` - Tests
- `apps/payload/src/seed/seed-engine/validators/data-validator.test.ts` - Tests
- `apps/payload/src/seed/seed-engine/validators/post-seed-validator.test.ts` - Tests

### Validation Rules
**Pre-Seeding**:
1. All JSON files parse correctly
2. All referenced collections exist in Payload config
3. All `{ref:}` patterns are syntactically valid
4. Required fields present in all records

**Post-Seeding**:
1. Record counts match expectations
2. Sample relationship integrity (check 10% of relationships)
3. Lexical content parseable
4. No orphaned relationships in `*_rels` tables
