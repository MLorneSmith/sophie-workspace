# Task: Integrate with Supabase reset and add npm scripts

## Description
Integrate the seeding infrastructure with the existing Supabase reset workflow and add comprehensive npm scripts for easy execution. This makes seeding a seamless part of the development workflow.

## Acceptance Criteria
- [ ] npm scripts added to `apps/payload/package.json`
- [ ] Integration with Supabase reset command working
- [ ] Optional `--seed` flag added to reset command
- [ ] All scripts properly documented
- [ ] Scripts work on Linux, macOS, and Windows (WSL)
- [ ] Error handling for common issues

## Technical Details
### Implementation Approach
1. Add npm scripts to `apps/payload/package.json`:
   ```json
   {
     "scripts": {
       "seed:run": "tsx src/seed/seed-engine/index.ts",
       "seed:dry": "tsx src/seed/seed-engine/index.ts --dry-run",
       "seed:validate": "tsx src/seed/seed-engine/index.ts --dry-run --verbose",
       "seed:courses": "tsx src/seed/seed-engine/index.ts -c courses,course-lessons"
     }
   }
   ```
2. Update `.claude/commands/database/supabase-reset.md`:
   - Add `--seed` flag to reset command
   - Run seeding after migrations if flag present
   - Update documentation
3. Test integration with existing workflows

### Files to Modify
- `apps/payload/package.json` - Add scripts
- `.claude/commands/database/supabase-reset.md` - Add --seed flag
- Update any related documentation

### Integration Points
```bash
# New workflow

# Equivalent to:
pnpm supabase:web:reset && pnpm --filter payload seed:run
```
