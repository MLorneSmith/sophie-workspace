# Task: Implement Payload initializer and base utilities

## Description
Create the Payload Local API initializer with singleton pattern and implement base utilities for logging and error handling. This provides the core infrastructure needed by all other seeding components.

## Acceptance Criteria
- [ ] Payload initializer created with singleton pattern
- [ ] Environment variable validation implemented
- [ ] Logger utility created with chalk-based colored output
- [ ] Base error handler created with retry logic
- [ ] Graceful cleanup on completion implemented
- [ ] All functions have comprehensive JSDoc comments
- [ ] Unit tests passing with >90% coverage

## Technical Details
### Implementation Approach
1. Create `payload-initializer.ts` with singleton pattern:
   - Single `getPayload()` call reused throughout
   - Load minimal seeding config (`payload.seeding.config.ts`)
   - Validate required environment variables
   - Handle initialization errors gracefully
2. Create `logger.ts` with colored output:
   - Support INFO, DEBUG, WARN, ERROR levels
   - Use chalk for colored terminal output
   - Support verbose mode flag
3. Create `error-handler.ts` with retry logic:
   - Exponential backoff with jitter
   - Classify retryable vs non-retryable errors
   - Maximum 3 retry attempts

### Files to Create
- `apps/payload/src/seed/seed-engine/core/payload-initializer.ts` - Payload API initialization
- `apps/payload/src/seed/seed-engine/utils/logger.ts` - Colored logging utility
