# Task: Implement batch Docker container health checks

## Description
Implement efficient batch health checking for all Docker containers using a single Docker API call. This optimization reduces overhead from N individual calls to 1 batch call, significantly improving performance while gathering comprehensive health data for all running containers.

## Acceptance Criteria
- [ ] Single docker inspect call retrieves all container health data
- [ ] JSON parsing handles containers with and without HEALTHCHECK
- [ ] Container categorization implemented (app, database, service)
- [ ] Performance target met: < 2 seconds for up to 50 containers
- [ ] Handles malformed or missing health data gracefully
- [ ] Works with both docker and docker-compose containers

## Technical Details
### Implementation Approach
1. Use `docker ps --format json` to list all containers
2. Batch retrieve health with `docker inspect` using container IDs
3. Parse JSON output (with jq if available, grep/sed fallback)
4. Categorize containers by inspecting labels and names
5. Build health status data structure for caching
