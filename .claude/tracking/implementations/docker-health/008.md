# Task: Add progressive health check strategies

## Description
Implement a multi-strategy health checking system that progressively checks container health using: Native Docker HEALTHCHECK → Port availability → Process status. This ensures comprehensive health coverage for containers with and without built-in health checks.

## Acceptance Criteria
- [ ] Native Docker HEALTHCHECK status retrieved when available
- [ ] Port availability check for containers exposing ports
- [ ] Process status check as final fallback
- [ ] Intelligent strategy selection based on container config
- [ ] Custom health check for known services (postgres, redis, etc.)
- [ ] Health state properly categorized (healthy, starting, unhealthy)

## Technical Details
### Implementation Approach
1. Primary: Check State.Health.Status from docker inspect
2. Secondary: Use nc/timeout to check exposed ports
3. Tertiary: Verify State.Running and State.ExitCode
4. Add service-specific checks for databases
5. Implement health state mapping and normalization
