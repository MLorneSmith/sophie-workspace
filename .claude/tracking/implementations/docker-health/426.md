# Task: Implement background process management with PID tracking

## Description
Create a robust background process management system that runs health checks every 30 seconds without blocking the statusline. Implement PID tracking to prevent duplicate processes and ensure proper cleanup on exit to avoid zombie processes.

## Acceptance Criteria
- [ ] Background process spawns correctly and detaches
- [ ] PID file created and managed at `/tmp/.claude_docker_health_pid_${GIT_ROOT}`
- [ ] Duplicate process prevention using PID checks
- [ ] Automatic cleanup of stale PID files
- [ ] Signal handling for graceful shutdown
- [ ] Process runs at nice level 10 to minimize impact
- [ ] No zombie processes created

## Technical Details
### Implementation Approach
1. Implement start_background_monitor() function
2. Create PID file with exclusive locking
3. Add is_process_running() check using kill -0
4. Implement cleanup_background_process() for shutdown
5. Add trap handlers for EXIT, INT, TERM signals
6. Use nohup and disown for proper detachment
