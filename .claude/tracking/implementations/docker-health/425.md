# Task: Create multi-level caching mechanism with TTL

## Description
Implement a three-tier caching strategy (in-memory → file-based → stale fallback) to balance performance requirements (<100ms statusline updates) with data freshness. This sophisticated caching system will minimize Docker API calls while ensuring status information remains current.

## Acceptance Criteria
- [ ] In-memory cache with 30-second TTL implemented
- [ ] File-based cache with 90-second TTL for persistence
- [ ] Stale data fallback up to 5 minutes for resilience
- [ ] Cache invalidation on container list changes
- [ ] Cache hit ratio exceeds 90% during normal operation
- [ ] Cache operations complete within 10ms

## Technical Details
### Implementation Approach
1. Implement memory cache using shell variables/arrays
2. Add TTL checking with timestamp comparisons
3. Create file cache with JSON structure and metadata
4. Implement cache warming on startup
5. Add smart invalidation based on container changes

### Potential Issues

- Cache invalidation timing could cause stale data
