# Task: Support multiple Docker Compose stacks

## Description
Enhance the health monitoring to recognize and group containers by their Docker Compose stacks. This allows for stack-level health aggregation and provides better visibility when working with multiple compose projects simultaneously.

## Acceptance Criteria
- [ ] Detects Docker Compose project labels
- [ ] Groups containers by compose project
- [ ] Provides stack-level health summaries
- [ ] Handles non-compose containers appropriately
- [ ] Supports custom project names and directories
- [ ] Display format shows stack grouping when relevant

## Technical Details
### Implementation Approach
1. Extract compose labels: com.docker.compose.project
2. Build container-to-stack mapping
3. Aggregate health by stack
4. Format display to show: "🟢 docker (web:3/3, db:2/2)"
5. Handle mixed compose and standalone containers
