# Task: Implement status file management with locking

## Description

Create a robust status file management system with proper file locking to prevent race conditions. This system will manage the primary status file at `/tmp/.claude_docker_status_${GIT_ROOT}` and implement atomic writes to ensure data consistency across concurrent accesses.

## Acceptance Criteria

- [ ] Status file created with proper naming convention including git root
- [ ] File locking implemented to prevent race conditions
- [ ] Atomic write operations using temp file and rename
- [ ] Old status cleanup on startup
- [ ] Proper file permissions set (user read/write only)
- [ ] JSON format for status data structure

## Technical Details

### Implementation Approach

1. Create `init_status_file()` function with git root detection
2. Implement `write_status()` with flock for exclusive access
3. Add `read_status()` with shared lock for concurrent reads
4. Use temp file + mv for atomic writes
5. Implement status file rotation/cleanup logic

### Potential Issues

- /tmp might be cleared during runtime

### Mitigation Strategies

- Recreate status file if missing
- Use PWD as fallback for git root
