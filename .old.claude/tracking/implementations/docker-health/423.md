# Task: Add statusline integration with emoji indicators

## Description

Integrate Docker health monitoring into the Claude Code statusline by adding the display logic to statusline.sh. Implement emoji-based status indicators (🟢🟡🔴⚪⟳) that follow existing patterns and provide at-a-glance container health visibility.

## Acceptance Criteria

- [ ] Integration point added to statusline.sh
- [ ] Emoji indicators display correctly: 🟢 (healthy), 🟡 (warning), 🔴 (critical), ⚪ (offline), ⟳ (checking)
- [ ] Format follows pattern: "🟢 docker (5/5)" for aggregated status
- [ ] Respects existing statusline formatting and spacing
- [ ] Updates happen within 100ms latency requirement
- [ ] Gracefully handles missing status file

## Technical Details

### Implementation Approach

1. Add docker_status() function to statusline.sh
2. Read status from cache file with timeout
3. Parse JSON status and calculate aggregate health
4. Format output with appropriate emoji and counts
5. Handle edge cases (no docker, no containers, errors)

### Files to Modify

- `.claude/bin/statusline.sh` - Add Docker status integration
- `.claude/bin/docker-health-wrapper.sh` - Export status reading function
