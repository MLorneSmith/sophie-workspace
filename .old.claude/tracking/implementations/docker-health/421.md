# Task: Implement Docker daemon detection and availability checks

## Description

Add Docker daemon detection logic to the wrapper script to determine if Docker is available, running, and accessible. This includes handling various Docker environments (Docker Desktop, Docker Engine, WSL2) and providing appropriate status indicators when Docker is unavailable.

## Acceptance Criteria

- [ ] Detects if Docker is installed on the system
- [ ] Checks if Docker daemon is running and accessible
- [ ] Handles permission errors gracefully with clear messages
- [ ] Supports Docker Desktop, Docker Engine, and WSL2 environments
- [ ] Returns appropriate status codes for different states
- [ ] Completes daemon check within 500ms

## Technical Details

### Implementation Approach

1. Implement `check_docker_daemon()` function using `docker version`
2. Add timeout wrapper to prevent hanging on unresponsive daemon
3. Detect Docker installation type (Desktop vs Engine)
4. Handle WSL2-specific Docker Desktop integration
5. Implement permission check with sudo detection
