# Task: Implement JSON data loader with validation

## Description

Create the JSON data loading module that reads seed data files from the filesystem, validates their structure, and returns typed collection data. This module provides the raw data for the seeding process.

## Acceptance Criteria

- [ ] JSON loader implemented with file system operations
- [ ] Parse error handling with line numbers
- [ ] Required field validation implemented
- [ ] Type validation for critical fields
- [ ] Graceful handling of missing files
- [ ] Support for loading all collections or specific collections
- [ ] Unit tests passing with >90% coverage

## Technical Details

### Implementation Approach

1. Create `json-loader.ts` with methods:
   - `loadCollection(name: string)` - Load single collection
   - `loadAllCollections()` - Load all collections
   - `validateStructure(data)` - Validate JSON structure
2. Implement error handling:
   - JSON parse errors with line numbers
   - Missing file errors with clear messages
   - Invalid structure errors with details
3. Add validation for required fields:
   - Check for `slug` or identifier field
   - Validate critical field types
   - Report validation errors clearly

### Files to Create

- `apps/payload/src/seed/seed-engine/loaders/json-loader.ts` - JSON loading logic
- `apps/payload/src/seed/seed-engine/loaders/json-loader.test.ts` - Unit tests

### Data Sources

Load from `apps/payload/src/seed/seed-data/`:

- `courses.json` - 1 record
- `course-lessons.json` - 25 records
- `course-quizzes.json` - 1 quiz
- `quiz-questions.json` - 94 questions
- `surveys.json` - ~10 surveys
- `survey-questions.json` - 246 questions
- `posts.json` - Blog posts
- `documentation.json` - Help docs
- `media-references.json` - 33+ media files
- `download-references.json` - 4+ downloads

## Risk Assessment

### Potential Issues

- JSON files may be malformed
- Large JSON files may cause memory issues
- File paths may differ across environments

### Mitigation Strategies

- Comprehensive validation before processing
- Stream large files if memory becomes an issue
- Use relative paths from project root
- Clear error messages with file names
