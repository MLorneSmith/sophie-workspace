# Task: Create collection processors

## Description

Implement the base processor abstract class and specialized processors for different collection types. Processors handle the creation of records via Payload Local API, with collection-specific logic where needed.

## Acceptance Criteria

- [ ] Base processor abstract class created
- [ ] Content processor implemented for generic collections
- [ ] Downloads processor implemented with UUID preservation
- [ ] Pre-process, process, and post-process hooks working
- [ ] Metadata removal working (`_ref`, `_status`)
- [ ] Payload API integration working correctly
- [ ] Unit tests passing with >90% coverage

## Technical Details

### Implementation Approach

1. Create `base-processor.ts` abstract class:

   ```typescript
   abstract class BaseProcessor {
     async preProcess(records: SeedRecord[]): Promise<void>
     async processRecord(record: SeedRecord): Promise<string>
     async postProcess(results: ProcessorResult[]): Promise<void>
   }
   ```

2. Create `content-processor.ts` for generic collections:
   - Remove internal metadata fields
   - Call `payload.create()` with cleaned data
   - Return created record UUID
3. Create `downloads-processor.ts` for downloads collection:
   - Preserve pre-assigned UUIDs when present
   - Handle external URL references
   - Validate file paths for media
