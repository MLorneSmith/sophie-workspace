# Overview

Develop a comprehensive implementation guide for setting up Warp terminal to efficiently manage and work with multiple GitHub Codespaces simultaneously. This solution will enable developers to streamline their workflow when working with multiple codespaces, providing seamless switching, resource monitoring, and command execution across environments.

## Background & Motivation

### Current Challenges

- Managing multiple GitHub Codespaces through the web interface is cumbersome
- No efficient way to execute commands across multiple codespaces simultaneously
- Limited terminal multiplexing capabilities in browser-based environments
- Difficult to monitor resource usage across multiple codespaces
- Complex SSH key management for multiple environments

### Benefits of Warp Terminal Integration

- **Native Terminal Experience**: Full-featured terminal with advanced features
- **Session Management**: Persistent sessions with automatic reconnection
- **Multi-Environment Workflows**: Seamless switching between codespaces
- **Enhanced Productivity**: Warp's AI-powered features and modern terminal UX
- **Resource Monitoring**: Better visibility into codespace health and usage

# Prerequisites

- Warp terminal installed ([Download](https://www.warp.dev/))
- GitHub account with Codespaces access and appropriate billing setup
- SSH key configured for GitHub authentication
- zsh shell (default on macOS, installable on Linux)
- Basic familiarity with SSH configuration and terminal workflows

**Objective**: Configure SSH keys for seamless codespace authentication

**Steps**:

1. **Generate dedicated SSH key for codespaces** (if not already done):

   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/github_codespaces -C "your-email@example.com"
   ```

   ```bash
   # Add to GitHub: Settings > SSH and GPG keys > New SSH key
   ```

3. **Configure SSH agent**:

   ```bash
   # Add to ~/.zshrc or ~/.bashrc
   eval "$(ssh-agent -s)"
   ```

**Objective**: Create convenient connection scripts for each codespace

**Create `~/.local/bin/connect-codespace-main.sh`**:

```bash
#!/bin/zsh

CODESPACE_NAME="slideheroes-main"
SSH_HOST="slideheroes-main"

# Function to connect to specific codespace
connect_to_codespace() {
    echo "🚀 Connecting to $CODESPACE_NAME..."
    
    # Check if codespace is running
    if gh codespace list --json name,state | jq -r ".[] | select(.name==\"$CODESPACE_NAME\") | .state" | grep -q "Available"; then
        echo "✅ Codespace is running"
        ssh $SSH_HOST
    else
        echo "⚠️  Codespace is not running. Starting..."
        gh codespace start --codespace $CODESPACE_NAME
        echo "⏳ Waiting for codespace to be ready..."
        sleep 30
        ssh $SSH_HOST
    fi
}

# Execute connection
connect_to_codespace
```

### 2.2 Batch Connection Script

**Objective**: Connect to multiple codespaces in separate Warp tabs

**Create `~/.local/bin/connect-all-codespaces.sh`**:

```bash
#!/bin/zsh

# Configuration
CODESPACES=("slideheroes-main" "slideheroes-dev" "slideheroes-feature")
WARP_COMMANDS=()

# Function to check codespace status
check_codespace_status() {
    local codespace_name=$1
    gh codespace list --json name,state | jq -r ".[] | select(.name==\"$codespace_name\") | .state"
}

# Function to start codespace if needed
ensure_codespace_running() {
    local codespace_name=$1
    local state=$(check_codespace_status $codespace_name)
    
    if [[ "$state" != "Available" ]]; then
        echo "🔄 Starting $codespace_name..."
        gh codespace start --codespace $codespace_name
        return 1
    fi
    return 0
}

# Main execution
echo "🔍 Checking codespace statuses..."

for codespace in "${CODESPACES[@]}"; do
    if ensure_codespace_running $codespace; then
        echo "✅ $codespace is ready"
        # Open new Warp tab and connect
        osascript -e "tell application \"Warp\" to activate"
        osascript -e "tell application \"System Events\" to keystroke \"t\" using {command down}"
        osascript -e "tell application \"System Events\" to keystroke \"ssh ${codespace}\""
        osascript -e "tell application \"System Events\" to keystroke return"
        sleep 2
    else
        echo "⏳ $codespace is starting..."
    fi
done

echo "🎉 All available codespaces connected!"
```

### 2.3 Codespace Management Utilities

**Create `~/.local/bin/codespace-manager.sh`**:

```bash
#!/bin/zsh

# Colors for output
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
BLUE='\033[0;34m'
NC='\033[0m' # No Color

# Function to show codespace status
show_status() {
    echo "${BLUE}📊 Codespace Status Dashboard${NC}"
    echo "=================================="
    
    gh codespace list --json name,state,machineDisplayName,region,repositoryName | \
    jq -r '.[] | "Name: \(.name)\nState: \(.state)\nMachine: \(.machineDisplayName)\nRegion: \(.region)\nRepo: \(.repositoryName)\n---"'
}

# Function to stop all codespaces
stop_all() {
    echo "${YELLOW}⚠️  Stopping all codespaces...${NC}"
    gh codespace list --json name,state | \
    jq -r '.[] | select(.state=="Available") | .name' | \
    while read codespace; do
        echo "🛑 Stopping $codespace"
        gh codespace stop --codespace "$codespace"
    done
}

# Function to start specific codespace
start_codespace() {
    local name=$1
    if [[ -z "$name" ]]; then
        echo "${RED}❌ Please provide codespace name${NC}"
        return 1
    fi
    
    echo "${GREEN}🚀 Starting $name...${NC}"
    gh codespace start --codespace "$name"
}

# Function to delete inactive codespaces
cleanup_inactive() {
    echo "${YELLOW}🧹 Cleaning up inactive codespaces...${NC}"
    gh codespace list --json name,state,lastUsedAt | \
    jq -r --arg threshold "$(date -d '7 days ago' -Iseconds)" \
    '.[] | select(.state=="Shutdown" and .lastUsedAt < $threshold) | .name' | \
    while read codespace; do
        echo "🗑️  Deleting inactive codespace: $codespace"
        read "confirm?Are you sure? (y/N) "
        if [[ $confirm =~ ^[Yy]$ ]]; then
            gh codespace delete --codespace "$codespace"
        fi
    done
}

# Main command dispatcher
case "$1" in
    "status"|"s")
        show_status
        ;;
    "stop-all")
        stop_all
        ;;
    "start")
        start_codespace "$2"
        ;;
    "cleanup")
        cleanup_inactive
        ;;
    *)
        echo "Usage: $0 {status|stop-all|start <name>|cleanup}"
        echo "  status    - Show all codespaces status"
        echo "  stop-all  - Stop all running codespaces"
        echo "  start     - Start specific codespace"
        echo "  cleanup   - Remove inactive codespaces"
        ;;
esac
```

### 2.4 Aliases and Functions

**Add to `~/.zshrc`**:

```bash
# Codespace aliases
alias cs-status='codespace-manager.sh status'
alias cs-stop='codespace-manager.sh stop-all'
alias cs-cleanup='codespace-manager.sh cleanup'
alias cs-connect='connect-all-codespaces.sh'

# Individual codespace connections
alias cs-main='connect-codespace-main.sh'
alias cs-dev='connect-codespace-dev.sh'
alias cs-feature='connect-codespace-feature.sh'

# Quick SSH functions
cs-ssh() {
    if [[ -z "$1" ]]; then
        echo "Usage: cs-ssh <codespace-name>"
        gh codespace list --json name | jq -r '.[].name'
        return 1
    fi
    ssh "$1"
}

# Multi-codespace command execution
cs-exec() {
    if [[ -z "$1" ]]; then
        echo "Usage: cs-exec '<command>' [codespace1] [codespace2] ..."
        return 1
    fi
    
    local cmd="$1"
    shift
    local codespaces=("$@")
    
    if [[ ${#codespaces[@]} -eq 0 ]]; then
        # Default to all available codespaces
        codespaces=($(gh codespace list --json name,state | jq -r '.[] | select(.state=="Available") | .name'))
    fi
    
    for cs in "${codespaces[@]}"; do
        echo "🔄 Executing on $cs: $cmd"
        ssh "$cs" "$cmd"
        echo "---"
    done
}
```

**Parallel Test Execution**:

```bash
#!/bin/zsh
# parallel-test-runner.sh

CODESPACES=("slideheroes-main" "slideheroes-dev" "slideheroes-feature")
TEST_COMMAND="cd /workspace && pnpm test"

run_tests_parallel() {
    local pids=()
    
    for cs in "${CODESPACES[@]}"; do
        (
            echo "🧪 Running tests on $cs..."
            ssh "$cs" "$TEST_COMMAND" > "test-results-$cs.log" 2>&1
            echo "✅ Tests completed on $cs"
        ) &
        pids+=($!)
    done
    
    # Wait for all tests to complete
    for pid in "${pids[@]}"; do
        wait "$pid"
    done
    
    echo "📊 All tests completed. Results:"
    for cs in "${CODESPACES[@]}"; do
        echo "--- $cs ---"
        tail -n 5 "test-results-$cs.log"
    done
}
```

### 4.2 File Synchronization Strategies

**Objective**: Keep files synchronized across multiple codespaces

**rsync-based Synchronization**:

```bash
#!/bin/zsh
# sync-codespaces.sh

MAIN_CODESPACE="slideheroes-main"
TARGET_CODESPACES=("slideheroes-dev" "slideheroes-feature")
SYNC_PATHS=("/workspace/apps" "/workspace/packages" "/workspace/.env.example")

sync_to_codespaces() {
    for target in "${TARGET_CODESPACES[@]}"; do
        echo "🔄 Syncing $MAIN_CODESPACE -> $target"
        
        for path in "${SYNC_PATHS[@]}"; do
            rsync -avz -e ssh "$MAIN_CODESPACE:$path" "$target:$(dirname $path)/"
        done
        
        echo "✅ Sync to $target completed"
    done
}
```

**Git-based Synchronization**:

```bash
#!/bin/zsh
# git-sync-codespaces.sh

sync_git_across_codespaces() {
```

```bash
#!/bin/zsh
# resource-monitor.sh

monitor_codespace_resources() {
    echo "📊 Codespace Resource Monitor"
    echo "============================="
    
    gh codespace list --json name,state | jq -r '.[] | select(.state=="Available") | .name' | while read cs; do
        echo "🖥️  $cs:"
        ssh "$cs" "
            echo '  CPU Usage:' \$(top -bn1 | grep 'Cpu(s)' | awk '{print \$2}' | cut -d'%' -f1)%
            echo '  Memory Usage:' \$(free -m | awk 'NR==2{printf \"%.1f%%\", \$3*100/\$2}')
            echo '  Disk Usage:' \$(df -h /workspace | awk 'NR==2{print \$5}')
            echo '  Processes:' \$(ps aux | wc -l)
        "
        echo ""
    done
}

# Real-time monitoring
monitor_realtime() {
    while true; do
        clear
        monitor_codespace_resources
        echo "🔄 Refreshing in 30 seconds... (Ctrl+C to stop)"
        sleep 30
    done
}
```

```bash
#!/bin/zsh
# auto-cleanup.sh

cleanup_idle_resources() {
    local codespaces=($(gh codespace list --json name,state | jq -r '.[] | select(.state=="Available") | .name'))
    
    for cs in "${codespaces[@]}"; do
        echo "🧹 Cleaning up $cs..."
        ssh "$cs" "
            # Clear temporary files
            find /tmp -type f -atime +1 -delete 2>/dev/null || true
            
            # Clear npm cache
            npm cache clean --force 2>/dev/null || true
            
            # Clear Docker containers if any

            # Clear logs older than 7 days

        "
    done
}
```
