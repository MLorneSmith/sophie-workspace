## 📋 Overview

Configure GitHub environment protection rules for production deployments to ensure proper review and approval processes.

## 🎯 Objective

Set up production environment protection to require 2 reviewers before any production deployment can proceed, implementing proper security and change management controls.

## ✅ Setup Checklist

### Step 1: Access Environment Settings

- [ ] Go to <https://github.com/MLorneSmith/2025slideheroes/settings/environments>
- [ ] You should see environments list (or option to create new environment)

### Step 2: Configure "production" Environment

1. **Access Production Environment**
   - [ ] Click on "production" environment (create it if it doesn't exist)

2. **Set Protection Rules**
   - [ ] ✅ Enable "Required reviewers"
   - [ ] Set minimum reviewers to: **2**
   - [ ] Add approved reviewers:
     - [ ] @MLorneSmith (repository owner)
     - [ ] Add other trusted team members

3. **Configure Deployment Branches**
   - [ ] ✅ Enable "Deployment branches"
   - [ ] Select "Selected branches"
   - [ ] Add rule for: `main` (only allow from main branch)

### Step 3: Configure "production-payload" Environment

1. **Access Production-Payload Environment**
   - [ ] Click on "production-payload" (create if it doesn't exist)

2. **Apply Same Protection Rules**
   - [ ] ✅ Enable "Required reviewers"
   - [ ] Set minimum reviewers to: **2**
   - [ ] Add same approved reviewers as production
   - [ ] Configure deployment branches (main only)

### Step 4: Optional Advanced Settings

**Deployment Windows** (if desired):

- [ ] Enable "Restrict deployments to specific time windows"
- [ ] Set allowed times (e.g., weekdays 9 AM - 5 PM)
- [ ] Configure appropriate timezone

**Environment Secrets** (if needed):

- [ ] Add production-specific secrets
- [ ] `VERCEL_AUTOMATION_BYPASS_SECRET`
- [ ] Other production-only variables

### Step 5: Verification

Test the protection rules:

- [ ] Create a test workflow dispatch for production
- [ ] Verify it enters "Waiting for approval" state
- [ ] Confirm only authorized users can approve
- [ ] Check that 2 approvals are required

## 📝 Important Notes

1. **UI-Only Configuration**: These settings cannot be configured via workflow YAML files - must use GitHub UI
2. **Immediate Effect**: Protection rules apply immediately once saved
3. **Team Communication**: Notify team before enabling to avoid deployment surprises
4. **Break-Glass Process**: Document emergency deployment procedures

- ✅ `production-deploy.yml` uses `environment: production-payload` for Payload
