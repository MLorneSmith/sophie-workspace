## Summary

During the production build process, a warning is logged indicating that git hash environment variables are not found. The application falls back to using the git command directly, which works but generates a warning. This affects the version endpoint's ability to efficiently retrieve the current git hash.

## Environment

- **Application Version**: 2.13.0
- **Environment**: production build
- **Node Version**: v18.18.0+
- **Build System**: Next.js with Turbo
- **Last Working**: N/A (non-breaking warning)

## Reproduction Steps

1. Run `pnpm build` in the project root
2. Observe the warning during the static page generation phase
3. Warning appears specifically when generating the `/version` route

## Expected Behavior

The build should retrieve the git hash from environment variables without warnings.

## Actual Behavior

Build completes successfully but displays:


### Code Analysis

The warning originates from `/apps/web/app/version/route.ts:54-56` where the application checks for git hash in these environment variables:

- `CF_PAGES_COMMIT_SHA` (Cloudflare Pages)
- `VERCEL_GIT_COMMIT_SHA` (Vercel)
- `GIT_HASH` (Custom)

When none are found, it falls back to executing `git log --pretty=format:"%h" -n1` directly.

## Initial Analysis

The warning is benign and doesn't affect functionality. The git hash is successfully retrieved via the fallback mechanism. However, this approach:

1. Is less efficient than reading from environment variables
2. May not work in containerized deployments without git
3. Creates unnecessary log noise during builds

## Suggested Investigation Areas

1. **Option 1: Add GIT_HASH to Build Process**
   - Set `GIT_HASH=$(git log --pretty=format:"%h" -n1)` before building
   - Add to `.claude/statusline/build-wrapper.sh`

2. **Option 2: Configure for Deployment Platform**
   - If using Vercel: Variable is auto-populated in production
   - If using Cloudflare Pages: Variable is auto-populated
   - For local/other platforms: Set `GIT_HASH` manually

3. **Option 3: Suppress Warning in Local Builds**
   - Only show warning in CI/production environments
   - Check for CI environment variables before logging

4. **Option 4: Static Git Hash at Build Time**
   - Generate git hash at build time and embed it
   - Avoid runtime git command execution entirely
