**Location**: `turbo.json` configuration

**Issue**: The Turbo configuration expects output files from the `@kit/monitoring-newrelic#test` task, but none are being generated. This suggests either:

## Proposed Solutions

### Fix 1: TSConfig Path Resolution

**Option A**: Fix the relative path

```json
// apps/payload/src/seed/seed-conversion/tsconfig.json
{
  "extends": "../../../../../tsconfig.json"  // Correct path to root tsconfig
}
```

**Option B**: Use the payload-specific tsconfig

```json
// apps/payload/src/seed/seed-conversion/tsconfig.json
{
  "extends": "../../../tsconfig.json"  // Extend from apps/payload/tsconfig.json
}
```

**Option C**: Remove if not needed
If this seed-conversion directory doesn't contain TypeScript files that need compilation, consider removing the tsconfig.json file entirely.

