## Summary

The E2E smoke test for security headers is failing because the development server is not returning the expected security headers (`x-frame-options`, `x-content-type-options`, and `referrer-policy`). This test expects these headers to be present on all responses but they are missing in the development environment.

## Environment

- **Application Version**: 0.1.0
- **Environment**: development
- **Browser**: Chromium (Playwright)
- **Node Version**: 22.16.0
- **Next.js**: 15.3.4
- **Current Branch**: dev
- **Related Issue**: Split from #121

## Reproduction Steps

1. Start the development server: `pnpm dev`
2. Run the smoke tests: `pnpm playwright test tests/smoke/smoke.spec.ts --grep "security headers"`
3. Test fails with "Received: undefined" for all three security headers

## Expected Behavior

The following security headers should be present on HTTP responses:

- `x-frame-options`: Prevents clickjacking attacks
- `x-content-type-options`: Prevents MIME type sniffing
- `referrer-policy`: Controls referrer information sent with requests

## Actual Behavior

All three security headers are missing from the response, causing the test to fail with:

```
Error: expect(received).toBeTruthy()
Received: undefined
```

## Test Code

The failing test is in `apps/e2e/tests/smoke/smoke.spec.ts`:

```typescript
test('security headers are present', async ({ request }) => {
  const response = await request.get('/');
  const headers = response.headers();

  // Check for important security headers
  expect(headers['x-frame-options']).toBeTruthy();
  expect(headers['x-content-type-options']).toBeTruthy();
  expect(headers['referrer-policy']).toBeTruthy();
});
```

## Initial Analysis

1. **Development vs Production**: Security headers might only be configured for production builds
2. **Next.js Configuration**: Headers may need to be added to `next.config.mjs`
3. **Middleware**: Security headers could be missing from the middleware configuration
4. **Test Environment**: The test might need to be adjusted for development environment expectations

## Suggested Investigation Areas

1. Check `next.config.mjs` for security header configuration
2. Review middleware setup for header injection
3. Determine if security headers should be present in development
4. Consider if the test should be skipped in development environment
5. Look for environment-specific header configuration
