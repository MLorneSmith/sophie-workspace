# Test Command Background Execution Prevents Progress Visibility

**ID**: ISSUE-352

## Summary

The `/test` command executes the test-controller.cjs script in the background, preventing users from seeing real-time test progress, results, and interactive feedback. This creates poor developer experience compared to direct pnpm test commands that provide live progress updates.

## Environment

- **Application Version**: 2.13.1
- **Environment**: development
- **Command Interface**: Claude Code slash commands
- **Test Framework**: Custom test-controller.cjs (538 lines)
- **Affected Command**: `/test` and all its variants (`--debug`, `--e2e`, etc.)

## Reproduction Steps

1. Run `/test --debug` command
   - ❌ **Result**: Command executes test-controller.cjs in background
   - ❌ **Behavior**: No live progress, limited user feedback
   - ❌ **Output**: Summary only after completion/timeout

2. Compare with direct pnpm command: `cd apps/e2e && pnpm test`
   - ✅ **Result**: Live progress with [1/132], [2/132] counters
   - ✅ **Behavior**: Real-time test execution feedback
   - ✅ **Output**: Immediate visibility into test status and failures

3. Run `/test --e2e` vs `pnpm test:e2e`
   - `/test --e2e`: Background execution, delayed/limited feedback
   - `pnpm test:e2e`: Live execution with immediate progress visibility

## Expected Behavior

The `/test` command should provide:

- **Real-time progress**: Live test execution updates
- **Interactive feedback**: Ability to see tests as they run
- **Immediate visibility**: Progress counters ([X/Y]), test names, pass/fail status
- **Error reporting**: Immediate visibility into failures as they occur
- **User control**: Ability to cancel or interact during execution

## Actual Behavior

The `/test` command currently:

- Executes test-controller.cjs as background process
- Provides minimal progress updates
- Shows results only after completion or timeout  
- Prevents user interaction during test execution
- Creates black-box experience vs transparent direct commands
