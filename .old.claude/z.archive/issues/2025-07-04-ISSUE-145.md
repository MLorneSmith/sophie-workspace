## Summary

Supabase security advisor has detected 18 PostgreSQL functions across the `public` and `payload` schemas with mutable search paths. This is a security concern where functions don't have their search_path parameter set to an empty string, potentially allowing schema poisoning attacks where malicious objects in other schemas could be executed instead of intended ones.

## Security Impact

The mutable search path vulnerability could allow:

1. **Schema Poisoning**: Attackers could create malicious objects in other schemas that would be executed instead of intended functions
2. **Privilege Escalation**: Functions running with elevated privileges (SECURITY DEFINER) are particularly vulnerable
3. **Data Manipulation**: Malicious functions could intercept and modify data flows

## Expected Behavior

All functions should have their search_path explicitly set to prevent schema poisoning attacks. This is typically done by setting `search_path = ''` in the function definition.

## Actual Behavior

Functions are created without explicit search_path settings, making them vulnerable to schema-based attacks.
