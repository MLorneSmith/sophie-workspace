## Issue Summary

The Dev Integration Tests workflow is consistently failing during dependency installation with a 403 Forbidden error when attempting to fetch pnpm from the npm registry.

## Root Cause Analysis

### Error Details

```
ERR_PNPM_FETCH_403  GET https://registry.npmjs.org/pnpm: Forbidden - 403

This error happened while installing a direct dependency of /home/runner/setup-pnpm

No authorization header was set for the request.
No authorization settings were found in the configs.
```

### Technical Analysis

The failure occurs in the `pnpm/action-setup@v4` action when it attempts to self-install pnpm. The npm registry is returning a 403 Forbidden error, indicating either:

1. Rate limiting from npm registry due to high CI/CD traffic
2. Missing or expired authentication tokens for private registry access
3. Network policy changes on GitHub Actions runners

### Contributing Factors

- The workflow runs from the `main` branch but tests the `dev` deployment
- The `setup-deps` action already includes retry logic with exponential backoff for the main installation, but the pnpm self-installer doesn't have similar protection
- No npm authentication token is configured for the GitHub Actions environment

## Recommended Fix

### Immediate Mitigation

1. Add npm registry authentication token to GitHub Secrets:

   ```yaml
   - name: Setup npm auth
     run: echo "//registry.npmjs.org/:_authToken=\${NPM_TOKEN}" >> ~/.npmrc
     env:
       NPM_TOKEN: \${{ secrets.NPM_TOKEN }}
   ```

2. Update the pnpm action setup to use a specific version instead of self-install:

   ```yaml
   - uses: pnpm/action-setup@v4
     with:
       version: 10.14.0  # Match package.json packageManager field
   ```

### Long-term Solution

1. Implement caching for pnpm binary to reduce registry calls
2. Add retry logic around the pnpm action setup
3. Consider using GitHub's built-in package registry mirror
4. Implement fallback to alternative installation methods

## Prevention Measures

1. **Add monitoring**: Set up alerts for CI/CD authentication failures
2. **Implement redundancy**: Multiple fallback installation methods
3. **Cache aggressively**: Cache pnpm binaries between workflow runs
4. **Rate limit protection**: Add delays between parallel job starts
5. **Documentation**: Document all required secrets and tokens
