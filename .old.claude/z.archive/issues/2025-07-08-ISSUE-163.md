## Summary

Snyk's free tier limit for private repositories has been exhausted, causing security scans to fail in our CI/CD pipeline with the error "You have used your limit of private tests". This is blocking our PR validation workflow and preventing proper security vulnerability scanning. We need to replace Snyk with an alternative that has a more generous free tier for private repositories.

## Environment

- **Application Version**: Current dev branch (18f86bbe)
- **Environment**: CI/CD Pipeline (GitHub Actions)
- **Affected Workflow**: PR Validation, Security Scans
- **Last Working**: Unknown - limit exhausted
- **Impact**: All PRs failing security checks

## Current Behavior

1. PR validation workflow runs Snyk security scan
2. Snyk returns: "You have used your limit of private tests"
3. Security scan job fails
4. PRs cannot be merged without admin override

## Root Cause

Snyk's free tier limitations for private repositories:

- 200 tests per month
- 10 tests per day
- Once exhausted, no more scans until next billing period

## Proposed Solution: Aikido Security

After researching alternatives, **Aikido Security** appears to be the best replacement:

### Why Aikido Security?

1. **Generous Free Tier**
   - Much more generous limits for small teams
   - Supports private repositories
   - No daily test limits

2. **Feature Parity (and more)**
   - SCA (Software Composition Analysis) for dependencies
   - SAST (Static Application Security Testing)
   - Secret scanning
   - License compliance
   - Malware detection (not in Snyk free tier)
   - SBOM generation

3. **Easy Integration**
   - GitHub Actions support via Local Scanner
   - Code scanned locally, only results uploaded
   - No GitHub Enterprise required
   - Works with standard GitHub accounts

4. **Privacy-First**
   - Local scanning means code never leaves GitHub Actions
   - Only scan results sent to Aikido platform

### Alternative Options Considered

| Tool                         | Pros                    | Cons                                               |
| ---------------------------- | ----------------------- | -------------------------------------------------- |
| **GitHub Advanced Security** | Native integration      | Requires GitHub Enterprise for private repos       |
| **Dependabot**               | Free, already available | Only dependency updates, no comprehensive scanning |
| **Socket.dev**               | Good for npm packages   | Limited scope compared to Snyk                     |
| **Mend.io**                  | Good SCA                | SAST requires paid tier                            |
| **SonarQube Community**      | Free and open source    | Limited SCA capabilities                           |

## Implementation Plan

1. **Immediate Action**
   - Comment out Snyk from workflows to unblock PRs
   - Enable Dependabot as temporary measure

2. **Aikido Integration**
   - Sign up for Aikido Security account
   - Generate API token
   - Add `AIKIDO_TOKEN` to GitHub secrets
   - Update workflows to use Aikido GitHub Action

3. **Workflow Changes**
   Replace in `.github/workflows/pr-validation.yml`:

   ```yaml
   # OLD
   - name: Run Snyk to check for vulnerabilities
     uses: snyk/actions/node@master
     env:
       SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}

   # NEW
   - name: Run Aikido Security Scan
     uses: aikidosec/github-action@v1
     with:
       token: ${{ secrets.AIKIDO_TOKEN }}
   ```

4. **Testing**
   - Test on feature branch first
   - Verify scan results quality
   - Ensure no false positives
