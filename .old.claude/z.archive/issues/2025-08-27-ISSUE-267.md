## Problem Statement

The test orchestrator can get stuck in "running" state indefinitely, requiring manual intervention. Recent debugging of Priority 2 issue revealed that tests appeared to hang for 3+ hours due to resource contention, not command generation issues.

### Root Cause Analysis

- **Resource contention**: 9 parallel E2E shards overwhelming dev server (34+ second response times)
- **Timeout cascade**: Playwright 20s timeouts exceeded by server response times
- **No orchestrator-level timeouts**: System waits indefinitely for results that never arrive
- **Parallel execution overload**: 9 shards × 4 workers = 36 concurrent browsers

### Impact

- Tests appear to hang indefinitely
- Manual cleanup required
- Development workflow disruption
- Unpredictable test execution timing

#### 1. Add Orchestrator-Level Timeouts

**Priority: Critical**

Update `.claude/agents/test/test-orchestrator.md` to include:

```javascript
const ORCHESTRATOR_TIMEOUT = 25 * 60 * 1000; // 25 minutes max
const E2E_PHASE_TIMEOUT = 20 * 60 * 1000;    // 20 minutes max for E2E
const UNIT_PHASE_TIMEOUT = 5 * 60 * 1000;    // 5 minutes max for unit tests

async function delegateWithTimeout(agentType, prompt, timeout) {
  return Promise.race([
    Task({ subagent_type: agentType, prompt }),
    new Promise((_, reject) => 
      setTimeout(() => reject(new Error(`Agent ${agentType} timeout after ${timeout}ms`)), timeout)
    )
  ]);
}
```
