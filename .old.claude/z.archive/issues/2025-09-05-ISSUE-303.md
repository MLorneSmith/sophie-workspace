## Issue Summary

The CI/CD pipeline for the `dev` branch is failing during the Pre-deployment Validation job due to malformed runner configuration syntax in multiple GitHub Actions workflows.

## Root Cause

The workflows are using an invalid `runs-on` syntax that GitHub Actions cannot parse:

```yaml
runs-on: runs-on=${{ github.run_id }},runner=2cpu-linux-x64
```

This syntax appears to be attempting to use a custom runner configuration but is not valid for GitHub Actions. The correct syntax should either be:

- Standard GitHub-hosted runners: `runs-on: ubuntu-latest`
- Self-hosted runners with labels: `runs-on: [self-hosted, linux, x64]`
- Matrix strategy for runners

## Affected Workflows

Based on analysis, the following workflows contain this malformed syntax:

- `.github/workflows/dev-deploy.yml`
- `.github/workflows/staging-deploy.yml` (likely)
- `.github/workflows/production-deploy.yml` (partially - some jobs use ubuntu-latest)

## Recommended Fix

### Immediate Fix (Option 1 - Use Standard Runners)

Replace all instances of the malformed syntax with standard GitHub-hosted runners:

```yaml
# Replace this:
runs-on: runs-on=${{ github.run_id }},runner=2cpu-linux-x64

# With this:
runs-on: ubuntu-latest
```

### Alternative Fix (Option 2 - Use Runner Labels for Different Sizes)

If different CPU configurations are needed, use GitHub's larger runners (requires GitHub Team/Enterprise):

```yaml
# For 2 CPU runners:
runs-on: ubuntu-latest  # Standard 2 CPU

# For 4 CPU runners (requires GitHub Team/Enterprise):
runs-on: ubuntu-latest-4-cores

# For 8 CPU runners (requires GitHub Team/Enterprise):
runs-on: ubuntu-latest-8-cores
```

### Alternative Fix (Option 3 - Self-Hosted Runners)

If custom runners are intended, use proper self-hosted runner syntax:

```yaml
runs-on: [self-hosted, linux, x64, 2cpu]
```
