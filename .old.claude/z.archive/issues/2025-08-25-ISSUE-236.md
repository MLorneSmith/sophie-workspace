## Problem Summary

The `/test` claude command is not working as designed, bypassing the test orchestration system and limiting E2E test visibility.

## Issues Identified

### 1. 🔴 Command Not Delegating to Test Orchestrator

**Current Behavior:**

- Running `claude test` executes tests directly without using test-orchestrator agent
- `claude test e2e` and `claude test all` incorrectly run only unit tests
- No parallel E2E execution happening

**Expected Behavior:**

- `/test` command should use Task tool to delegate to test-orchestrator agent
- test-orchestrator coordinates unit-test-agent and e2e-parallel-agent
- E2E tests run in 9 parallel shards for ~70% faster execution

### 2. ⚠️ Max-Failures Limiting Test Visibility

**Issue:** All test:shard commands have `--max-failures=1` flag
**Impact:** E2E suite stops after first failure (only 25/85 tests ran)
**Location:** `apps/e2e/package.json` lines with test:shard[1-9]

### 3. 🐛 Account Deletion Test Timeout

**Test:** `account.spec.ts:80:6 › Account Deletion › user can delete their own account`
**Error:** TimeoutError: page.waitForURL timeout after 20000ms waiting for navigation to "/"
**File:** `/apps/e2e/tests/account/account.spec.ts:88:14`

## Root Cause

The claude test command implementation needs to properly use the Task tool to delegate to test-orchestrator agent instead of directly executing tests.

## Recommended Fixes

### Priority 1: Fix Command Delegation

Update `/test` command to use:

```javascript
Task({
  subagent_type: "test-orchestrator",
  description: "Coordinate test execution",
  prompt: "Execute comprehensive test suite..."
})
```

### Priority 2: Remove max-failures Limit

Update `apps/e2e/package.json`:

- "test:shard1": "playwright test tests/accessibility/accessibility-hybrid.spec.ts --max-failures=1",

Apply to all 9 shard commands for full test visibility.
