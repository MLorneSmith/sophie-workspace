The CI/CD pipeline uses inconsistent deployment approaches across dev, staging, and production environments. Dev and production use the custom `vercel-deploy` action, while staging uses inline Vercel CLI commands. Additionally, only the dev environment deploys both Web and Payload CMS apps, while staging and production only deploy the Web app.

All environments should:

1. Use the standardized `vercel-deploy` action consistently

## Problems This Causes

3. **Maintenance Overhead**: Multiple deployment approaches to maintain
4. **Debugging Difficulty**: Different behaviors across environments
5. **Missing CMS in Production**: Payload CMS isn't deployed to production at all
