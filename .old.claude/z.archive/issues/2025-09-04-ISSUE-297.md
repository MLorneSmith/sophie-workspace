## Overview

Follow-up from #296. The current shard optimizer generates configurations that break test dependencies by redistributing tests based solely on complexity/time metrics, ignoring functional dependencies between test files.

## Current Issues

### 1. Test Dependencies Ignored

The optimizer's "balanced" strategy separates tests that must run together:

- Authentication tests separated from account tests
- Team setup separated from team billing tests  
- Admin tests missing entirely from generated config

### 2. Configuration Override

The optimizer overwrites the carefully designed `testGroups` configuration in `test-controller.cjs` with an auto-generated `shard-config.json` that doesn't respect test relationships.

### 3. Missing Test Files

Some test files (like `admin.spec.ts`) are not discovered or included in the generated shard configuration.

## Required Changes

### 1. Use testGroups as Source of Truth

```javascript
// test-shard-optimizer.cjs
class ShardOptimizer {
  async generateShardConfig(options) {
    // Start with testGroups from test-controller.cjs
    const baseGroups = this.loadTestGroups();
    
    // Only optimize within groups, not across them
    return this.optimizeWithinGroups(baseGroups, options);
  }
}
```

### 2. Add Dependency Mapping

```javascript
const testDependencies = {
  'auth.spec.ts': {
    requires: [],
    providesFor: ['account.spec.ts', 'team-accounts.spec.ts']
  },
  'account.spec.ts': {
    requires: ['auth.spec.ts'],
    providesFor: ['team-accounts.spec.ts']
  },
  'team-accounts.spec.ts': {
    requires: ['auth.spec.ts', 'account.spec.ts'],
    providesFor: ['team-billing.spec.ts', 'team-invitation-mfa.spec.ts']
  }
};
```

### 3. Make Optimization Opt-In

Change default behavior to use configured groups unless explicitly enabled:

```javascript
// test-controller.cjs line 1970
// Change from:
if (this.shardOptimizer && process.env.OPTIMIZE_SHARDS !== "false") {
// To:
if (this.shardOptimizer && process.env.OPTIMIZE_SHARDS === "true") {
```

### 4. Ensure All Tests Discovered

Add validation to ensure all test files are included:

```javascript
async validateShardConfig(config) {
  const allTestFiles = await this.discoverAllTests();
  const configuredTests = config.shards.flatMap(s => s.tests);
  
  const missing = allTestFiles.filter(f => !configuredTests.includes(f));
  if (missing.length > 0) {
    throw new Error(`Missing tests in shard config: ${missing.join(', ')}`);
  }
}
```
