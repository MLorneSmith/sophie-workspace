# Quiz ID Consistency Fix Implementation Plan (Part 3)

## Problem Analysis

The content migration process is failing with the following error:

```
Error in content processing migration: error: insert or update on table "quiz_questions" violates foreign key constraint "quiz_questions_quiz_id_fkey"
Key (quiz_id)=(c11dbb26-7561-4d12-88c8-141c653a43fd) is not present in table "course_quizzes".
```

This error indicates a mismatch between quiz IDs referenced in the quiz questions and the quiz IDs that exist in the course_quizzes table. Specifically, a quiz question is trying to reference a quiz with ID `c11dbb26-7561-4d12-88c8-141c653a43fd`, but this quiz doesn't exist in the `course_quizzes` table.

### Database Schema Analysis

The database has the following key tables and relationships:

1. `course_quizzes` - Contains the quizzes with a primary key `id`
2. `quiz_questions` - Contains questions with foreign keys `quiz_id` and `quiz_id_id` that reference `course_quizzes.id`
3. `course_lessons` - Contains lessons with optional foreign keys `quiz_id` and `quiz_id_id` that reference `course_quizzes.id`

The foreign key constraints ensure that:

- `quiz_questions.quiz_id` must exist in `course_quizzes.id`
- `quiz_questions.quiz_id_id` must exist in `course_quizzes.id`
- `course_lessons.quiz_id` must exist in `course_quizzes.id`
- `course_lessons.quiz_id_id` must exist in `course_quizzes.id`

### Current Implementation Analysis

The SQL seed files are generated by the `generate-sql-seed-files-fixed.ts` script, which:

1. Defines a `knownQuizIds` object that maps quiz slugs to UUIDs
2. Uses the `generateQuizMap` function to create a map of quiz slugs to UUIDs
3. Generates SQL files for courses, lessons, quizzes, and questions
4. Includes a `verifyQuizIds` function to check that the quiz IDs in the generated SQL match those in `knownQuizIds`

The `knownQuizIds` object correctly contains the ID `c11dbb26-7561-4d12-88c8-141c653a43fd` for `basic-graphs-quiz`, and this matches the ID in `quiz-id-map.json`. However, there's a discrepancy between the quiz IDs used in the `03-quizzes.sql` file and the `04-questions.sql` file.

## Root Cause Identification

After analyzing the code and error logs, I've identified several potential issues:

1. **SQL Generation Issue**: The `generateQuizzesSql` function might not be correctly using the IDs from the quiz map for all quizzes.

2. **Verification Limitation**: The `verifyQuizIds` function uses a regex pattern that might not correctly match all quiz IDs in the SQL, allowing inconsistencies to slip through.

3. **Cross-File Verification Gap**: There's no verification step to ensure that the quiz IDs used in the questions SQL file match those in the quizzes SQL file.

4. **Regex Pattern Issue**: The regex pattern in `verifyQuizIds` might not be correctly extracting the quiz IDs and slugs from the SQL, leading to false positives in the verification.

The most likely root cause is that the `generateQuizzesSql` function is not consistently using the IDs from the quiz map for all quizzes, particularly for the `basic-graphs-quiz`. This leads to a situation where the quiz ID in the `03-quizzes.sql` file doesn't match the expected ID `c11dbb26-7561-4d12-88c8-141c653a43fd` that's used in the `04-questions.sql` file.

## Proposed Solution

To fix the quiz ID consistency issue, we need to enhance the SQL generation process to ensure quiz IDs are consistent across all files:

1. **Fix the `verifyQuizIds` Function**:

   - Update the regex pattern to correctly match the SQL structure
   - Make the function more robust by checking all occurrences of quiz IDs in the SQL
   - Add checks to ensure all known quiz IDs are present in the SQL

2. **Enhance the `generateQuizzesSql` Function**:

   - Add additional logging to show which IDs are being used for each quiz
   - Ensure that the function is correctly using the IDs from the quiz map
   - Add validation to ensure that all quiz slugs have a corresponding UUID in the map

3. **Add a Cross-File Verification Step**:

   - After generating all SQL files, verify that the quiz IDs in `03-quizzes.sql` match those in `04-questions.sql`
   - If there's a mismatch, fail the generation process with a clear error message

4. **Implement a Fix for the Current Issue**:
   - Ensure that the ID for `basic-graphs-quiz` in `03-quizzes.sql` matches the expected ID `c11dbb26-7561-4d12-88c8-141c653a43fd`

### 1. Update the `generateQuizzesSql` Function

```typescript
function generateQuizzesSql(
  quizzesDir: string,
  quizMap: Map<string, string>,
): string {
  // Get all .mdoc files in the quizzes directory
  const quizFiles = fs
    .readdirSync(quizzesDir)
    .filter((file) => file.endsWith('.mdoc'));

  // Start building the SQL
  let sql = `-- Seed data for the course quizzes table
-- This file should be run after the courses seed file to ensure the course exists

-- Start a transaction
BEGIN;

`;

  // Process each quiz file
  for (const file of quizFiles) {
    const filePath = path.join(quizzesDir, file);
    const fileContent = fs.readFileSync(filePath, 'utf8');
    const { data } = matter(fileContent);

    // Get the quiz slug and UUID
    const quizSlug = path.basename(file, '.mdoc');
    const quizId = quizMap.get(quizSlug);

    if (!quizId) {
      console.error(`Error: No ID found for quiz ${quizSlug}`);
      continue;
    }

    console.log(`Generating SQL for quiz ${quizSlug} with ID ${quizId}`);

    // Add the quiz to the SQL
    sql += `-- Insert quiz: ${data.title}
INSERT INTO payload.course_quizzes (
  id,
  title,
  slug,
  description,
  passing_score,
  created_at,
  updated_at
) VALUES (
  '${quizId}', -- UUID for the quiz
  '${data.title.replace(/'/g, "''")}',
  '${quizSlug}',
  '${(data.description || `Quiz for ${data.title}`).replace(/'/g, "''")}',
  ${data.passingScore || 70}, -- Default passing score is 70
  NOW(),
  NOW()
) ON CONFLICT (id) DO NOTHING; -- Skip if the quiz already exists

`;
  }

  // End the transaction
  sql += `-- Commit the transaction
COMMIT;
`;

  return sql;
}
```

### 2. Fix the `verifyQuizIds` Function

```typescript
function verifyQuizIds(quizzesSql: string): boolean {
  const quizIdRegex =
    /INSERT INTO payload\.course_quizzes[\s\S]*?id = '([^']+)'[\s\S]*?slug = '([^']+)'[\s\S]*?/g;
  let match;
  let allMatch = true;
  const foundIds = new Map<string, string>();

  while ((match = quizIdRegex.exec(quizzesSql)) !== null) {
    const id = match[1];
    const slug = match[2];

    foundIds.set(slug, id);

    if (knownQuizIds[slug] && id !== knownQuizIds[slug]) {
      console.error(
        `Error: ID mismatch for quiz ${slug}. SQL has ${id}, knownQuizIds has ${knownQuizIds[slug]}`,
      );
      allMatch = false;
    }
  }

  // Check if all known quiz IDs are in the SQL
  for (const [slug, id] of Object.entries(knownQuizIds)) {
    if (!foundIds.has(slug)) {
      console.error(`Error: Quiz ${slug} with ID ${id} not found in SQL`);
      allMatch = false;
    }
  }

  return allMatch;
}
```

### 3. Add a Cross-File Verification Step

```typescript
function verifyCrossFileQuizIds(
  quizzesSql: string,
  questionsSql: string,
): boolean {
  // Extract quiz IDs from quizzes SQL
  const quizIdRegex =
    /INSERT INTO payload\.course_quizzes[\s\S]*?id = '([^']+)'[\s\S]*?/g;
  const quizIds = new Set<string>();
  let match;

  while ((match = quizIdRegex.exec(quizzesSql)) !== null) {
    const id = match[1];
    quizIds.add(id);
  }

  // Extract quiz IDs from questions SQL
  const questionQuizIdRegex = /quiz_id = '([^']+)'/g;
  const questionQuizIds = new Set<string>();

  while ((match = questionQuizIdRegex.exec(questionsSql)) !== null) {
    const id = match[1];
    questionQuizIds.add(id);
  }

  // Check if all question quiz IDs exist in quizzes
  let allExist = true;
  for (const id of questionQuizIds) {
    if (!quizIds.has(id)) {
      console.error(
        `Error: Quiz ID ${id} referenced in questions does not exist in quizzes`,
      );
      allExist = false;
    }
  }

  return allExist;
}
```
